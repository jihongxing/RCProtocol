# Task: Stage 5 MVP 可交付闭环落地

> **关联 Spec**: [spec-stage-5-mvp-delivery.md](../specs/spec-stage-5-mvp-delivery.md)  
> **状态**: ✅ 已完成  
> **优先级**: P0  
> **预计工期**: 5-8 天

---

## 任务目标

把 RCProtocol 当前已具备的协议内核、品牌注册、激活、验真、过户、前端页面与服务接入能力，收口为一个：

1. 可重复联调
2. 可稳定回归
3. 可最小部署
4. 可对外演示
5. 可支持品牌试点预演

的 MVP 交付闭环。

本任务不以继续扩展协议终局能力为目标，而以 **Stage 5 正式验收** 为目标。

---

## 审计说明

本任务用于推进当前 `roadmap.md` 中的：

- 5.1 Redis 投影与运维增强
- 5.2 全链路集成测试
- 5.3 品牌 API 对接联调
- 5.4 异常流覆盖
- 5.5 性能基线
- 5.6 部署方案落地
- 5.7 MVP 验收

为避免任务再次发散，本文档只拆当前最关键的可交付工作，不新增终局协议设计范围。

## Phase 2: 黄金主链路联调（1.5 ~ 2 天）

- [x] 记录链路中的人工步骤并尽量脚本化

**验收标准**:
- [x] 主链路在约定环境中可重复执行
- [x] 每一步均有明确成功判定标准

### Task 2.2: 收口 V1 / V2 验真并行语义

- [x] 明确两者的使用场景差异
- [x] 确认 V2 不破坏当前 MVP 演示路径
- [x] 对外演示文案中明确“标签真实性”和“承诺完整性”的区别

**验收标准**:
- [x] 同一资产可稳定得到 V1 / V2 响应

**验收标准**:
- [x] 主链路关键状态可被页面或接口稳定观察
- [x] 联调时不需要频繁直接查库才能判断是否成功
