# V5.0 全球实物资产数字主权协议 — 项目文档

---

## 一、项目总览

### 1.1 项目定位

V5.0 是一套基于 NFC 加密芯片（NTAG 424 DNA）的**全球实物资产数字主权协议**。通过独创的"母子标签纠缠"机制，为高价值实物资产（奢侈品、腕表、名酒、收藏品等）建立不可伪造的数字身份，实现真伪核验、所有权确权、二手流转清算的完整闭环。

项目的终极形态不是防伪工具，而是**实物资产的数字清算中心**——一个中立的、基于物理密码学的"实物资产交易所"。

### 1.2 核心价值主张

| 维度 | 内容 |
|------|------|
| 身份确定性 (ID) | 通过母子纠缠加密，证明"它是它" |
| 价值参考性 (Value) | 通过 V-Value 引擎，提供多维市场参考价 |
| 权利法律性 (Ownership) | 通过数字过户，明确"它是谁的" |

### 1.3 三方解耦原则

系统采用"你管安全，厂家管内容"的极致解耦：

- **平台方（你）**：仅存储 UID、AES 密钥、物理状态和品类指针 (KID)，掌握清算与仲裁权
- **品牌方**：通过 API 维护产品图文、视频及营销权益，掌握资产定义权
- **解析器 (Resolver)**：用户扫码时实时聚合两端数据，呈现"真伪结论 + 品牌内容"

### 1.4 核心比喻

> 母卡是"印钞机的模具"，子卡是"钞票"。工厂只负责造纸（子卡），平台给品牌方发放模具（母卡），品牌方拿着模具在纸上盖章（点睛），最后出来的才是真正的货币（资产）。

#### 收益层 (The Revenue Engine)

## 三、核心协议规范

### 3.1 母子纠缠协议 (The Entanglement Protocol)

#### 核心逻辑

资产的激活必须满足三要素：**[母卡权限] + [子卡芯片] + [云端指令] = 资产确权**

#### 纠缠流程

1. **物理纠缠**：品牌方点睛人员先感应母卡（持有权证明），再感应子卡（资产本体）
2. **数字加签**：App 将母卡签名信息与子卡 UID 结合发送给 KMS
3. **云端闭环**：KMS 校验母卡合法性、配额充足性，下发加密指令完成绑定

#### 状态机定义

每一组母子映射拥有严格的生命周期状态：

| 状态码 | 名称 | 含义 |
|--------|------|------|
| 0 | PRE_MINTED | 已生产，标签已出厂但尚未进入供应链 |
| 1 | FACTORY_LOGGED | 已盲扫，工厂已植入，物理来源合法，无数字身份 |
| 2 | ACTIVATED | 已激活，母子纠缠完成，商品正式上线 |
| 3 | LEGALLY_SOLD | 合法售出，消费者完成确权 |
| 4 | TRANSFERRED | 已过户，二手交易完成权属变更 |
| 5 | CONSUMED | 已消耗，资产被正常使用殆尽（如名酒开瓶、雪茄点燃），物理标签失效，数字主权转入荣誉态，不可流转 |
| 6 | LEGACY | 遗珍，资产物理损坏不可修复，数字主权以"残缺美"形式永久保留，记录主权存续时长 |
| 7 | TAMPERED | 已破坏，子标签物理回路断裂，永久作废 |
| 8 | COMPROMISED | 已失陷，检测到克隆或异常攻击 |
| 9 | DESTRUCTED | 已注销，品牌方主动执行资产报废 |

### 3.2 盲扫协议 (Blind-Entry Protocol)

#### 设计原则：物理生产黑盒化、资产定义中心化

**第一步：平台发放"空白授权"**
- 向工厂下发空白子标签，数据库中只有 UID，无任何品牌/SKU 属性
- 对工厂而言，这些标签只是没有任何信息的电子零件

**第二步：工厂端盲扫植入**
- 工厂每生产一个商品，扫描空白子标并贴入
- 工厂后台仅发送：`UID_12345 已在工厂 A 生产线被扫描`
- 工厂端系统不显示任何商品名称、SKU 或图片，操作员只看到"扫描成功"

**第三步：品牌方母卡激活与身份赋予**
- 商品运抵品牌方总部仓库后，总部人员用母标签扫描子标签
- 系统验证子标来源合法 → 品牌方选择秘密 SKU → 执行母子纠缠 → 身份注入
- 此刻芯片从"无名氏"变成"正品"

#### 漏洞封堵

| 风险场景 | 盲扫协议的防御 |
|----------|---------------|
| 工厂多造产品 | 工厂只有子标签，没有母标签无法赋予身份 |
| 新品泄密 | 工厂系统只有 UID，工人不知道产品是什么 |
| 标签被盗 | 偷走的只是空白 UID，品牌方不授权则无法激活 |

### 3.3 链式授权协议 (Chain-Linked Activation)

权力由上至下物理传递，每一级必须获得上一级的"物理感应 + 云端授权"方可生效：

1. **生产预埋**：工厂将空白子卡缝入资产，资产处于 [待激活] 状态
2. **渠道赋能**：总部通过 Genesis Tag 感应 Channel Tag，分配合法化点数
3. **终端销售激活**：消费者付款后，柜员用 Store Tag 感应 Asset Tag，资产状态变为 LEGALLY_SOLD
4. **消费者确权**：消费者扫码完成所有权登记
5. **二次流转**：二手交易时通过"双扫"完成资产过户

### 3.4 权力分配：双重否决制

| 角色 | 责任 | 执行方式 |
|------|------|----------|
| 品牌方 | 主张权 — 判定产品质量、发往市场、销售对象 | 通过物理母卡发起点睛请求 |
| 平台方 | 仲裁权 — 验证母卡合法性、配额充足性、下发指令 | 通过 KMS 校验并下发加密指令 |

## 四、安全体系

### 4.1 KMS 密钥管理系统

#### 分层衍生体系 (KDF)

绝不在数据库明文存储芯片密钥，使用密钥衍生函数：

1. **根密钥 (Master Root Key)**：存储在物理加密机 (HSM) 中，任何人无法读取
2. **品牌派生密钥 (Brand Key)**：由 Root_Key 结合 Brand_ID 派生，每个品牌独立密钥空间
3. **芯片终端密钥 (Leaf Key / K2)**：由 Brand_Key 结合 UID 派生
   - 公式：`K_chip = HMAC-SHA256(Brand_Key, UID)`
   - 数据库只需存 UID，不需要存密钥

#### 核心组件

| 组件 | 功能 |
|------|------|
| 安全执行环境 (TEE) | 所有加密运算在隔离内存中完成，对操作系统不可见 |
| 动态指令生成器 | 接收 UID → 验证配额 → 生成 K2 → 封装加密 APDU 指令流 |
| 确权认证网关 | 接收 UID+CTR+CMAC → 派生 K2 → 对比 CMAC → 发放 JWT |

#### 熔断机制

- **计数器回溯拦截**：CTR ≤ 历史记录 → 判定克隆卡攻击，拉黑请求
- **频率限制**：单个 UID 1分钟内查询超10次 → 冻结验证功能
- **地理围栏校验**：5分钟内跨洲扫码 → 触发风控标记

### 4.2 母卡防克隆保护协议 (Anti-Cloning Protocol)

#### 三位一体防伪架构

1. **物理层 — PUF 芯片**：利用半导体生产中微小物理差异，每颗芯片有唯一"硅指纹"，无法克隆
2. **算法层 — 双向动态握手**：App 发送随机数 → 母卡内部计算 HMAC → 主权密钥永远不出芯片
3. **策略层 — 滚动机理**：母卡内置不可逆计数器，云端检测到流水号回溯或不连续跳变 → 全局熔断

#### 分级管理体系

| 级别 | 名称 | 持有者 | 核心权限 |
|------|------|--------|----------|
| L0 | 创世母卡 (Genesis) | 品牌方 CEO / 首席法务 | 最高主权：激活和授权下级母卡；重置品牌根密钥 |
| L1 | 区域母卡 (Regional) | 区域负责人 / 供应链总监 | 分拨权：在授权额度内分配给工厂或仓库 |
| L2 | 点睛母卡 (Operator) | 一线质检员 / 仓库管理员 | 操作权：执行具体的子卡点睛 |

#### 防丢失与挂失流程

- **丢失预防**：App 必须检测母卡物理存在 + 操作员生物特征，双重验证后才下发指令
- **紧急挂失 (Kill Switch)**：管理员在 Sovereign Console 点击挂失 → KMS 将母卡公钥加入黑名单 → 全球失效
- **主权修复**：L0 创世母卡丢失 → 需品牌方 CEO + 技术总监 + 平台方三方数字私钥多签认证 → 生成新根密钥

### 4.3 设备端安全接入

#### 双层证书体系

- **L1 生产级设备 (Factory Reader)**：仅具备"盲扫"权限，证书权限受限且具备过期机制
- **L2 核心级设备 (Brand Activator)**：具备"纠缠与定义"权限，需绑定品牌方硬件安全密钥

#### 通讯安全

- **mTLS 双向认证**：设备证书 + 私钥签名 + 订单令牌三要素
- **HMAC 校验**：每条 API 请求 Body 包含基于设备私钥的签名
- **时钟同步**：强制时间戳校验（误差 ±30s），防止离线录入后批量重放

#### 证书生命周期

- **任务制证书**：生产订单下达时激活，订单完成或扫描数量达上限后自动吊销
- **紧急吊销**：监控发现异常高频扫描 → OCSP 实时阻断设备通讯

### 4.4 全链路加密传输

- **通讯层**：强制 mTLS，每个终端内置独一无二的设备证书
- **数据层**：AES-256-GCM 对 API body 二次加密，设备私钥签名 + 平台公钥加密
- **密钥派生**：`K_tag = CMAC(K_master, UID ∥ System_ID)`，单芯片被破解不影响全局
- **SUN 消息**：芯片每次扫码用 AES-128 产生动态密文，含自增计数器防重放

### 4.5 隐私主权架构

#### 核心原则："零知识、物理隔离、主权自持"

系统在技术架构上主动放弃"窥探权"，以赢得用户的"信任权"。对于高净值人群，"被看见"比"被偷走"更令其恐惧。

#### 零知识验证

- 系统可以验证"某人拥有某资产"，但系统本身不知道"此人是谁"
- 用户生物信息仅保留在本地设备，云端存储经过哈希脱敏的匿名权属令牌
- 即便数据库被攻破，攻击者只能看到匿名 ID 与资产的映射关系，无法对应到自然人

#### 地理位置"阅后即焚"

- 扫码验证时，地理位置仅用于风控触发，验证通过后立即物理抹除空间坐标
- 系统不记录用户带着资产去了哪里，只证明资产在持有人手中

#### 主权冷存储

- 核心权属密钥不在云端，而在用户手中的母标签硬件中
- 即便平台停止运营，只要用户持有母标签，仍可通过去中心化节点证明资产归属
- 平台定位为"数字化镜像服务商"，不持有用户资产

### 5.1 芯片选型

| 角色 | 型号 | 部署位置 | 核心功能 |
|------|------|----------|----------|
| 子标签 (Child) | NTAG 424 DNA TagTamper | 物品主体（易碎贴/隐蔽处） | 状态监控：物理回路感应，一撕即报废 |
| 母标签 (Mother) | NTAG 424 DNA 标准版 | 藏家卡/保修卡（随身保管） | 权限控制：作为资产所有权钥匙 |

### 5.2 NTAG 424 DNA 核心参数

#### 安全参数
- **AES-128 硬件加密引擎**：支持 5 个可独立配置的密钥
- **SUN (Secure Unique NFC) 消息认证**：在不打开加密通道的情况下，直接在 URL 里生成动态密文

#### 存储与协议
- **内存容量**：416 Bytes，经硬件安全加固
- **标准**：ISO/IEC 14443-A & NFC Forum Type 4 Tag，兼容所有 iPhone 和 Android

#### 物理形态建议
- **天线尺寸**：建议直径 20mm 以上
- **电容值**：建议 50pF，调谐稳定性好，适配皮质/布料材质，感应距离 1-3cm

#### 采购形态

| 形态 | 适用场景 | 备注 |
|------|----------|------|
| 空白芯片 (Wafer/Dry Inlay) | 工业化大规模嵌入 | 最便宜，需自有贴合工艺 |
| 湿嵌体 (Wet Inlay) | 透明不干胶形式 | 首选，自带背胶，适合前期阶段 |
| 定制标签 (Finished Tag) | 高端品牌定制 | PVC卡片、皮质挂件，成本略高 |

### 5.3 SDM 配置 (Secure Dynamic Messaging)

#### 密钥分配

- **K0 (Master Key)**：用于修改配置
- **K1 (SDM Meta Key)**：用于加密 UID（可选）
- **K2 (SDM File Read Key)**：用于生成动态 CMAC（核心）

#### 动态 URL 结构

```
https://v5.auth/verify?uid=04A1B2C3D4E5F6&ctr=000005&cmac=8A3F...
```

- `uid`：芯片全球唯一 7 字节序列号（明文）
- `ctr`：芯片内部 3 字节 NFC 计数器，每感应一次 +1
- `cmac`：由芯片使用内部密钥对 uid 和 ctr 进行 AES-128 计算生成的动态密文

#### APDU 配置流程

1. **认证与选卡**：使用初始密钥进行 AES 认证
2. **配置 SDM 动态参数镜像**：启用 SDM，定义 CTR 和 CMAC 在 URL 中的偏移位置
3. **写入基础 URL**：写入带占位符的 H5 链接，位置与 Offset 精确对应
4. **锁定计数器**：开启 Mirror CTR，确保每次感应触发 CMAC 更新
5. **最终锁定**：修改 K0 并锁定配置区（不可逆）

### 5.4 物理封装策略

- **子标签**：超薄 + 易碎天线 + 强力背胶，设计"弱连带"结构使天线回路跨越商品缝隙，剥离即断裂
- **母标签**：标准 PVC 卡片或金属镭雕卡，不需要防拆，需极高耐用性和品牌质感

## 六、API 接口规范

### 6.1 全局逻辑对象

- **C-UID (Child UID)**：子标签物理唯一标识，代表"物品实物"
- **M-UID (Mother UID)**：母标签物理唯一标识，代表"所有权/定义权"
- **SKU-ID**：品牌方定义的数字资产身份，代表"商品灵魂"

### 6.2 阶段一：工厂端 — 盲扫入库

**接口**：`POST /v1/factory/blind-log`

**请求参数**：
```json
{
  "factory_auth_key": "STRING",
  "c_uid_sun": "STRING",
  "timestamp": "LONG",
  "batch_no": "STRING"
}
```

**平台逻辑**：
1. 校验 factory_auth_key 合法性
2. 解密 c_uid_sun 提取 C-UID
3. 检查该 C-UID 是否已存在
4. 状态变更为 FACTORY_LOGGED

**返回**：
```json
{
  "code": 200,
  "msg": "LOG_SUCCESS",
  "data": { "trace_id": "UUID" }
}
```

> 严禁返回任何商品名称或品牌信息

### 6.3 阶段二：品牌总部 — 纠缠激活

**接口**：`POST /v1/brand/entangle-active`

**请求参数**：
```json
{
  "brand_auth_token": "STRING",
  "m_uid_sun": "STRING",
  "c_uid_sun": "STRING",
  "target_sku_id": "STRING"
}
```

**平台逻辑**：
1. 验证 C-UID 是否处于 FACTORY_LOGGED 状态
2. 验证 M-UID 是否为该品牌下发的合法母卡
3. 建立 M-UID ↔ C-UID ↔ SKU-ID 三位一体映射
4. 状态变更为 ACTIVATED

**返回**：
```json
{
  "code": 200,
  "msg": "IDENTITY_GRANTED",
  "data": {
    "activation_time": "ISO8601",
    "asset_status": "LIVE"
  }
}
```

### 6.4 阶段三：消费者端 — 动态解析

**接口**：`GET /v1/resolve/{sun_msg}`

**解析逻辑**：

| 状态 | 返回内容 |
|------|----------|
| FACTORY_LOGGED | 品牌方预设的"生产中/认证中"占位页 |
| ACTIVATED | 302 重定向至品牌方 SKU 详情页，携带加密 Verify_Token |
| LEGALLY_SOLD | 正品认证 + 持有人证书 + 多维行情 |
| TAMPERED | 红色报警页（物理防伪破损） |

### 6.5 后端校验逻辑（伪代码）

```python
def verify_v5_tag(uid, ctr, cmac_from_url):
    # 1. 获取该 UID 对应的 K2
    k2 = get_key_by_uid(uid)

    # 2. 检查计数器是否异常（防重放）
    if int(ctr) <= last_recorded_ctr(uid):
        return "REPLAY_ATTACK_DETECTED"

    # 3. 构造计算素材
    sv = construct_sdm_vector(uid, ctr)

    # 4. AES-128 CMAC 计算
    expected_cmac = aes_cmac(k2, sv)

    # 5. 比对
    if cmac_from_url == expected_cmac:
        update_last_ctr(uid, ctr)
        return "SUCCESS_GENUINE_ASSET"
    else:
        return "COUNTERFEIT_DETECTED"
```

### 6.6 安全审计规则

- **Anti-Replay**：每次调用校验 SUN 消息中的计数器，计数值 ≤ 历史值 → 判定克隆攻击
- **Geo-Fencing**：盲扫接口记录工厂 IP 范围，非授权经纬度 → 标记 SUSPICIOUS
- **Entropy Check**：某母卡短时间内纠缠大量子标 → 封锁该 M-UID

### 7.4 荣誉陈列室 (Honor Display Room)

#### 设计理念

主权不随物理消亡而终结，它只是从"交易态"转入"荣誉态"。喝掉的酒、背坏的包不应从数字藏馆中消失——它们是用户消费权力与生活轨迹的勋章，如同老派贵族保留的猎鹿角、收藏家珍藏的空酒瓶。

#### 资产终极状态三态模型

| 资产状态 | 物理情况 | 数字体现 | 社交价值 |
|----------|----------|----------|----------|
| Active (活跃) | 完好，可使用 | 动态光效，显示实时估值 | 即时炫耀：证明持有且可交易 |
| Consumed (已消耗) | 酒已喝，烟已抽 | 置灰并加盖皇家火漆印，显示开启日期 | 历史勋章：证明消费层级 |
| Legacy (遗珍) | 物理损坏，不可修复 | 残缺美视觉效果，记录主权存续时长 | 情怀叙事：曾经拥有的老战友 |

#### 核心机制

1. **不可剥夺的消费履历**：物理标签可碎、酒瓶可空，但 V5 协议记录的"你曾在某年某月拥有并开启了这瓶酒"的记录永恒存在，无法伪造

2. **空瓶数字主权证书**：名酒用户执行"确认开启"操作后，物理标签失效（防止二次灌装造假），云端资产自动移入荣誉陈列室，手机中永久保留该资产的数字主权证书

3. **消费累积分级 (Consumption Tiering)**：系统统计用户消耗历史，生成消费等级排名（如"已累计消耗 50 瓶飞天茅台，击败全球 99% 酒友"），消耗产生的等级感比单纯持有更具社交杀伤力

#### 品牌方长尾价值

- **物理断后防伪**：资产标记为"已消耗"后，即便标签仍在也无法再进入二级市场流转，从源头杜绝回收真瓶灌假酒的产业链
- **情感锁定复购**：用户看着藏馆中一排"已消耗"勋章，产生极强的品牌忠诚度和复购欲望

## 九、用户体系

### 9.1 账户模型："1 + N" 模式

- **唯一 ID (UID)**：每个用户一个唯一的 V5 系统 ID
- **双重身份属性**：
  - 个人模式 (Personal)：收藏、炫耀、个人流转
  - 主体模式 (Entity/Company)：品牌方、经销商、二奢店
- **数字签名 (Private Key)**：流转确权本质是原持有人"所有权放弃签名" + 新持有人"接收确认签名"

### 9.2 匿名身份锚点（最小化数据占用）

身份简化为"匿名手机号 + 物理母标签 + 生物特征"的分布式信任网络：

- **匿名手机号**：仅作为单向脱敏的验证码接收通道，数据库存储 SHA-256 加盐哈希值，无法反向推导真实号码
- **品牌方隔离**：品牌方在后台只能看到资产流转状态，绝对看不到用户手机号
- **匿名中转通讯**：品牌方联系持有人必须通过 V5 匿名中转平台，品牌方发消息给平台，平台根据哈希值找到对应通道转发
- **随时断开权**：用户可随时更换绑定的匿名号码，资产根主权锚定在母标签和生物识别上，号码只是临时"数字外壳"

| Owner_Asset_Portfolio | 持有人资产组合 | 卖给私人银行、豪车品牌的顶级数据 |

### 10.3 消费与荣誉态字段

| 字段 | 描述 | 商业价值 |
|------|------|----------|
| Consumption_State | 资产终极状态 (Active/Consumed/Legacy) | 区分可交易资产与荣誉态资产 |
| Consumption_Date | 消耗确认日期 | 消费履历时间线 |
| Consumption_Tier | 用户消耗累积等级 | 消费分级排名与社交展示 |
| Honor_Display_Rank | 荣誉陈列室展示排序 | 藏馆个性化展示 |

### 10.4 交易行为特征

| 字段 | 描述 | 商业价值 |
|------|------|----------|
| Clearing_Event_Log | 清算事件记录 | 分账审计 |
| Appreciation_Depreciation_Rate | 资产溢价/折价率 | 反映品牌保值率 |
| Authentication_Frequency | 验证频率（未过户） | 高频 = 正在被多个买家询价 |

### 10.5 社交与关联关系

| 字段 | 描述 | 商业价值 |
|------|------|----------|
| Circle_Transfer_Link | 圈层流转链 | 勾勒高净值社交圈 |
| Cross_Brand_Affinity | 跨品牌关联度 | 交叉销售数据 |

### 10.6 芯片内属性码 (4字节 Sovereign Descriptor)

在点睛时写入芯片的极简属性码：

```
品类码(1B) + 品牌系列码(1B) + MSRP(2B)
示例：02-01-AF40 → [包类]-[香奈儿系列]-[45000元]
```

扫码时芯片返回：`UID + Category:02 + Grade:SS + MSRP:45000`，即使外部行情失效也能定价。

## 十一、实施路线图

### 11.1 阶段一：种子期（0-3个月）— 秘密构建数字清算引擎

**技术收口**：
- 固化底层 v1.0 算法，完成 NTAG 424 DNA 动态密文解析模块
- 部署 KMS 至多地 HSM
- 开发盲扫入库接口（不带 UI 的后端 API）
- 编写母子配对演示逻辑
- 开发动态重定向测试（未激活/已激活/报警三种状态）

**硬件验证**：
- 采购 NTAG 424 DNA 和 TagTamper 样品
- 验证易碎天线在不同材质上的粘贴效果
- 跑通 SDM 配置与动态 URL 校验

**市场准备**：
- 定制无 Logo 空白智能芯片
- 开发简易"品牌数字资产管理系统" App
- 寻找 1-2 个国内新兴高端设计师品牌试点

### 11.2 阶段二：渗透期（3-12个月）— 建立官方过户习惯

- 协议层预留 Asset_Type 字段，支持多品类横向兼容
- 正式激活"过户认证"功能和分润模块
- 给头部二奢线下店提供"免费验证终端"
- 话术转变：以"品牌方额外收入来源"推介过户费
- 部署二奢真品流转平台（RC Certified Pre-Owned），实现"挂牌即确权"零人工鉴定模式
- 建立 `v1/market/escrow-lock`（交易锁）接口，支持资金隔离与原子化过户

### 11.3 阶段三：爆发期（12个月+）— 成为实物世界的结算标准

- 跨行业迁移：名酒（50元/次）、潮玩/限量鞋（19元/次）
- 数据资产化：输出高净值人群实物资产分布图
- 成立"全球实物资产数字化标准组织"(V5 Alliance)
- 形成"无 V5.0 不交易"的市场共识

### 11.4 从物理卡到 App 的演进路线

| 阶段 | 模式 | 特点 |
|------|------|------|
| 初期 | H5 + 全物理卡 | 极高安全性，部署极快，不依赖 App |
| 中期 | 混合模式 | 生产端保留创世母卡，渠道/零售端下放 App |
| 成熟期 | 全数字化主权 | 手机就是母卡，所有纠缠在云端和手机硬件层完成 |

系统设计时将物理卡 (Tag_ID) 和 App 账户 (User_ID) 视为同等地位的"权限载体"，保持向下兼容。

### 11.5 深水区补丁

**1. 司法级证据链**：每次点睛和过户同步推送摘要至去中心化存证链，支持法庭举证

**2. 资产死亡协议**：
- 资产报废：母卡感应子卡执行 SET_DESTRUCTED，KMS 永久注销 UID
- 品牌方破产：协议具备"托管模式"，已确权用户仍可查询真伪

**3. 跨国贸易清算**：
- 底层以 SDR 或稳定币计价
- H5 根据用户 IP 自动切换币种
- 支持全球分账的支付网关

## 十三、核心术语表

| 术语 | 定义 |
|------|------|
| 母子纠缠 (Entanglement) | 母标签与子标签通过加密算法建立的不可伪造的物理绑定关系 |
| 点睛 (Activation) | 品牌方使用母卡触碰子卡，赋予资产数字身份的操作 |
| 盲扫 (Blind-Entry) | 工厂端在不知道商品信息的情况下扫描并植入空白子标签 |
| 子弹 (Quota) | 品牌方向平台购买的点睛配额 |
| SUN (Secure Unique NFC) | NTAG 424 DNA 的动态消息认证功能 |
| SDM (Secure Dynamic Messaging) | 安全动态消息，使 URL 中的参数在每次感应时自动变化 |
| CMAC | 基于 AES-128 的消息认证码，用于验证芯片真伪 |
| KMS | 密钥管理系统，协议的核心安全中枢 |
| HSM | 硬件安全模块，物理隔离的加密机 |
| TEE | 可信执行环境，隔离内存中的安全运算空间 |
| PUF | 物理不可克隆函数，利用芯片物理差异生成唯一指纹 |
| Sovereign Vault | 数字私人藏品馆 |
| Sovereign Console | 品牌方/平台方管理后台 |
| Sovereign Fee | 动态确权认证费 |
| V-Value | 多维价值看板/实时估值引擎 |
| Honor Display Room | 荣誉陈列室，已消耗/遗珍资产的永久数字展示空间 |
| Consumed (已消耗) | 资产终极状态之一，物理资产被正常使用殆尽，数字主权转入荣誉态 |
| Legacy (遗珍) | 资产终极状态之一，物理资产损坏不可修复，数字主权以残缺美形式永久保留 |
| Consumption Tiering | 消费累积分级，基于用户消耗历史生成的消费等级排名体系 |
| Privacy Manifesto | 隐私宣言，V5.0 协议的数据主权哲学宣言，核心原则为"零知识、物理隔离、主权自持" |
| Zero-Knowledge Proof | 零知识证明，系统可验证资产归属但不知道持有人真实身份的技术架构 |
| Sovereign Cold Storage | 主权冷存储，核心权属密钥存于用户母标签硬件而非云端的安全策略 |
| Data Minimization | 最小化数据占用，将身份锚点简化为匿名手机号+母标签+生物特征的设计原则 |
| Silent Airdrop | 静默空投，品牌方在用户藏品馆内无感投放营销内容的隐私友好触达方式 |
