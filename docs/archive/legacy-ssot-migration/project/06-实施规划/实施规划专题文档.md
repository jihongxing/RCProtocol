## 五、技术实施五阶段

### 5.1 硬件标准与物理形态定型

- 子标签工业设计：锁定芯片、定制易碎天线、打样测试不同材质适配性
- 母标签视觉设计：确定为标准 PVC 证书卡或金属镭雕卡

### 5.2 云端安全引擎与解析器开发

- KMS 密钥管理系统：基于 AES-128 的动态密钥生成算法
- 状态机与解析器：识别 SDM 密文，自动区分状态，动态拼装内容
- 反欺诈与假货地图模块：时空审计模型，验证失败数据漏斗

### 5.3 品牌方对接协议与 API 库

- 制定元数据推送规范
- 开发 Vendor Dashboard（品牌方管理后台）
- 提供 SDK/H5 模板库

### 5.4 产线联合铸造方案部署

- 开发初始化设备写码软件
- 制定 SOP：物理绑定 + 数字激活
- 防超发审计：记录每一组铸造行为

### 5.5 试点验证与市场化推进

- 封闭内测：选择 1-2 个友好品牌小规模实测
- 获客数据回测：统计扫码用户入会率与推送点击率
- 商业白皮书发布：基于实测数据启动行业招商

## 六、从物理卡到 App 的演进路线

| 阶段 | 模式 | 特点 |
|------|------|------|
| 初期 | H5 + 全物理卡 | 极高安全性，部署极快，不依赖 App |
| 中期 | 混合模式 | 生产端保留创世母卡，渠道/零售端下放 App |
| 成熟期 | 全数字化主权 | 手机就是母卡，所有纠缠在云端和手机硬件层完成 |

---

## 七、避坑指南

1. **不挑战现状**：不说取代中检，说是品牌方给消费者的额外数字福利
2. **轻量化部署**：不要求工厂改流水线，芯片贴在吊牌还是缝在内衬由品牌方决定
3. **利益均沾**：给品牌方分润是项目不被掐死的唯一保障，让他们觉得这是自己的项目
