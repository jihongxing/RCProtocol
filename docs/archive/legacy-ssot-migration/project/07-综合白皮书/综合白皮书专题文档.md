## 一、项目核心定位

V5.0 是全球首个基于母子纠缠算法的实物资产数字中央银行。项目的终极形态不是防伪工具，而是实物资产的数字清算中心——一个中立的、基于物理密码学的实物资产交易所。

### 1.1 核心价值主张

| 维度 | 内容 |
|------|------|
| 身份确定性 (ID) | 通过母子纠缠加密，证明它是它 |
| 价值参考性 (Value) | 通过 V-Value 引擎，提供多维市场参考价 |
| 权利法律性 (Ownership) | 通过数字过户，明确它是谁的 |

### 1.2 市场痛点

- 全球高净值实物资产存量超 10 万亿美元
- 每年因超发和二奢造假导致的品牌损失超 500 亿美元
- 品牌方无法追踪商品在二级市场的流转

## 三、API 接口规范

### 3.1 全局逻辑对象

- **C-UID (Child UID)**：子标签物理唯一标识，代表物品实物
- **M-UID (Mother UID)**：母标签物理唯一标识，代表所有权/定义权
- **SKU-ID**：品牌方定义的数字资产身份，代表商品灵魂

### 3.2 阶段一：工厂端盲扫入库

接口：`POST /v1/factory/blind-log`

请求参数：factory_auth_key, c_uid_sun, timestamp, batch_no

平台逻辑：校验工厂授权 -> 解密提取 C-UID -> 检查是否已存在 -> 状态变更为 FACTORY_LOGGED

严禁返回任何商品名称或品牌信息。

### 3.3 阶段二：品牌总部纠缠激活

接口：`POST /v1/brand/entangle-active`

请求参数：brand_auth_token, m_uid_sun, c_uid_sun, target_sku_id

平台逻辑：验证 C-UID 状态 -> 验证 M-UID 合法性 -> 建立三位一体映射 -> 状态变更为 ACTIVATED

### 3.4 阶段三：消费者端动态解析

接口：`GET /v1/resolve/{sun_msg}`

| 状态 | 返回内容 |
|------|----------|
| FACTORY_LOGGED | 品牌方预设的生产中/认证中占位页 |
| ACTIVATED | 302 重定向至品牌方 SKU 详情页 |
| LEGALLY_SOLD | 正品认证 + 持有人证书 + 多维行情 |
| TAMPERED | 红色报警页 |

## 四、商业合作技术保障备忘录（面向品牌方）

### 4.1 三大技术保障支柱

1. **供应链黑盒隔离**：工厂仅接触空白授权标签，定义权上收至品牌总部
2. **母子纠缠算法**：AES-128-SUN 动态密文，防重放攻击，物理防拆
3. **全链路加密通讯**：mTLS 双向认证，HSM 物理隔离密钥

### 4.2 品牌方核心收益

- 获客成本降低 80% 以上
- 100% 实时假货监控覆盖
- 二手市场信用背书与过户分红

### 4.3 合规承诺

- 轻资产部署：品牌方无需改造生产线
- 数据脱敏：协议仅处理芯片状态与资产归属
- 主权归属：用户资产数据归属权始终属于品牌方

## 七、隐私宣言与数据主权哲学

### 7.1 隐私宣言 (The Privacy Manifesto)

> "在 V5 宇宙中，数据不属于平台，而属于主权者。我们看不见您的名字，听不见您的隐私，我们只为您守护那份不可篡改的尊严。"

V5.0 协议的隐私处理决定了项目的生死。走"卖数据换钱"的路是高级 CRM 软件；走"守护主权"的路，才是数字时代的瑞士银行。

### 7.2 隐私架构核心原则

| 原则 | 实现方式 | 效果 |
|------|----------|------|
| 零知识验证 | 云端仅存匿名权属令牌，生物信息留在本地 | 数据库泄露也无法对应自然人 |
| 阅后即焚 | 地理位置仅用于风控触发，验证后立即物理抹除 | 不记录用户轨迹 |
| 主权冷存储 | 核心权属密钥存于用户母标签硬件 | 平台倒闭不影响资产归属证明 |
| 最小化数据占用 | 身份锚点仅为匿名手机号+母标签+生物特征 | 分布式信任，无中心化身份库 |

### 7.3 数据使用哲学：不卖数据，卖基于数据的顶级服务

数据的使用必须是被动触发且隐形的：

1. **资产保值天气预报 (Invisible Consulting)**：不需要知道用户是谁，根据藏馆内资产分布推送宏观市场情报。话术不是"张总，你那包涨了"，而是"阁下，您持有的同类资产在全球拍卖市场溢价 15%，是否需要更新保险保额？"

2. **匿名入场券 (Identity as a Service)**：用户参加私人晚宴无需出示邀请函，只需展示匿名主权证明。V5 协议证明"此人持有 10 件以上博物馆级藏品"，通过身价筛选但保护真实姓名。

3. **品牌方盲盒营销**：品牌方不获取用户名单，只获得"空投通道"。品牌可针对特定条件（如"持有 V5 认证包 3 年以上且未流转"）的用户在藏馆内静默空投邀请函。用户感受是"品牌懂我"而非"数据监视我"。

### 7.4 品牌空投商业模型

| 类型 | 品牌方操作 | V5 收益点 |
|------|-----------|----------|
| 基础空投 (Standard) | 推送保养提醒、品牌历史故事 | 基础服务费（按触达资产数量计费） |
| 高级空投 (Premium) | 推送限量款配额、私人晚宴门票 | 分成模式：用户通过空投产生复购则参与抽成 |
| 定向空投 (Targeted) | 针对长期忠诚持有人精准空投 | 算法溢价：全球最精准的财富人群匹配费 |

### 7.5 品牌方隔离策略

- 品牌方在后台只能看到资产流转状态，绝对看不到用户联系方式
- 品牌方联系持有人必须通过 V5 匿名中转平台
- 用户主动授权前，品牌方无法获取任何个人信息

### 7.6 V5 空投的降维打击优势

- **绝对真实性**：邀请函只发给物理感应激活、通过生物识别、资产在藏品馆内的真实持有人，"以物找人"误差率为零
- **主权级直达**：品牌方可绕过门店销售等中间环节，一键空投至用户私人藏馆
- **极高心理转化率**：用户在私人藏馆看到品牌直接空投的邀请函，"被品牌点名"的荣誉感远超传统营销
