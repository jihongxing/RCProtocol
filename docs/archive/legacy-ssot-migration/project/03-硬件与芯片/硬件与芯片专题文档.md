## 二、NTAG 424 DNA 核心参数

### 2.1 安全参数

- **AES-128 硬件加密引擎**：支持 5 个可独立配置的密钥
- **SUN (Secure Unique NFC) 消息认证**：在不打开加密通道的情况下，直接在 URL 里生成动态密文

### 2.2 存储与协议

- **内存容量**：416 Bytes，经硬件安全加固
- **标准**：ISO/IEC 14443-A 与 NFC Forum Type 4 Tag，兼容所有 iPhone 和 Android

### 2.3 物理形态建议

- **天线尺寸**：建议直径 20mm 以上
- **电容值**：建议 50pF，调谐稳定性好，适配皮质/布料材质，感应距离 1-3cm

## 四、SDM 配置 (Secure Dynamic Messaging)

### 4.1 密钥分配

- **K0 (Master Key)**：用于修改配置
- **K1 (SDM Meta Key)**：用于加密 UID（可选）
- **K2 (SDM File Read Key)**：用于生成动态 CMAC（核心）

### 4.2 APDU 配置流程

1. **认证与选卡**：使用初始密钥进行 AES 认证
2. **配置 SDM 动态参数镜像**：启用 SDM，定义 CTR 和 CMAC 在 URL 中的偏移位置
   - FileOption: 0x40（启用 SDM）
   - AccessRights: 0x1211（定义 K2 为读取和认证密钥）
   - SDMOptions: 0x81（启用 ASCII 模式）
   - SDMCtrOffset / SDMMACRefOffset / SDMMACOffset：定义各字段在 URL 中的位置
3. **写入基础 URL**：写入带占位符的 H5 链接，位置与 Offset 精确对应
4. **锁定计数器**：开启 Mirror CTR，确保每次感应触发 CMAC 更新
5. **最终锁定**：修改 K0 并锁定配置区（不可逆）

### 4.3 动态 URL 结构

```
https://v5.auth/v?u=00000000000000&c=000000&m=0000000000000000
```

u (UID)、c (CTR)、m (MAC) 的位置必须与配置的 Offset 精确对应。

## 五、物理封装策略

- **子标签**：超薄 + 易碎天线 + 强力背胶，设计弱连带结构使天线回路跨越商品缝隙，剥离即断裂
- **母标签**：标准 PVC 卡片或金属镭雕卡，不需要防拆，需极高耐用性和品牌质感

## 六、芯片内属性码 (4字节 Sovereign Descriptor)

在点睛时写入芯片的极简属性码：

```
品类码(1B) + 品牌系列码(1B) + MSRP(2B)
示例：02-01-AF40 -> [包类]-[香奈儿系列]-[45000元]
```

扫码时芯片返回：`UID + Category:02 + Grade:SS + MSRP:45000`，即使外部行情失效也能定价。

## 八、开发工具推荐

### 8.1 桌面端（底层开发与批量写入）

- **硬件**：ACR122U 读写器
- **软件**：NFC 21 Tools（图形化 NDEF 写入）、NXP NTAG 424 DNA Tag Operations（SDM 配置专用）

### 8.2 移动端（随时随地测试）

- **NXP TagWriter**：官方工具，编写基础 URL 链接
- **NFC Tools Pro (Android)**：读取芯片详细信息，操作原始数据块
- **NXP TagInfo**：透视芯片所有技术参数

### 8.3 自研点睛 App

1. 扫一下：读取芯片唯一 UID
2. 算一下：后端生成该 UID 对应的唯一密钥
3. 写一下：将动态 URL 写入芯片并锁定
