# V5.0 架构与协议专题文档

---

## 一、协议总体架构

### 1.1 四层架构

| 层级 | 名称 | 核心职责 |
|------|------|----------|
| 物理层 | Physical Anchors | 子标签预埋实物，动态 CMAC 码；母标签作为物理钥匙 |
| 身份层 | Identity Core | 生物识别锁死账户；多因子验证 |
| 业务层 | Business Logic | 私人藏品馆、主权预览码、原子化交割 |
| 收益层 | Revenue Engine | 动态行情费、以旧换新策略 |

### 1.2 三方解耦原则

- **平台方**：仅存储 UID、AES 密钥、物理状态和品类指针 (KID)，掌握清算与仲裁权
- **品牌方**：通过 API 维护产品图文、视频及营销权益，掌握资产定义权
- **解析器**：用户扫码时实时聚合两端数据，呈现真伪结论与品牌内容

### 1.3 三方数据主权分布

| 实体 | 掌握的数据 |
|------|-----------|
| 平台 (KMS) | 芯片加密密钥、母子纠缠映射表、物理防拆状态、时空异常审计 |
| 品牌方 (CMS) | 商品名称、多媒体素材、营销直达 URL、会员权益 |
| 硬件载体 (NFC) | 动态加密 URL、唯一的物理 UID |

## 二、母子纠缠协议

### 2.1 核心逻辑

资产激活三要素：[母卡权限] + [子卡芯片] + [云端指令] = 资产确权

### 2.2 纠缠流程

1. **物理纠缠**：点睛人员先感应母卡（持有权证明），再感应子卡（资产本体）
2. **数字加签**：App 将母卡签名信息与子卡 UID 结合发送给 KMS
3. **云端闭环**：KMS 校验母卡合法性与配额充足性，下发加密指令完成绑定

### 2.3 状态机定义

| 状态码 | 名称 | 含义 |
|--------|------|------|
| 0 | PRE_MINTED | 已生产，标签已出厂但尚未进入供应链 |
| 1 | FACTORY_LOGGED | 已盲扫，工厂已植入，物理来源合法，无数字身份 |
| 2 | ACTIVATED | 已激活，母子纠缠完成，商品正式上线 |
| 3 | LEGALLY_SOLD | 合法售出，消费者完成确权 |
| 4 | TRANSFERRED | 已过户，二手交易完成权属变更 |
| 5 | CONSUMED | 已消耗，物理资产使用殆尽，数字主权转入荣誉态 |
| 6 | LEGACY | 遗珍，物理损坏不可修复，数字主权以残缺美形式永久保留 |
| 7 | TAMPERED | 已破坏，子标签物理回路断裂，永久作废 |
| 8 | COMPROMISED | 已失陷，检测到克隆或异常攻击 |
| 9 | DESTRUCTED | 已注销，品牌方主动执行资产报废 |

## 三、盲扫协议 (Blind-Entry Protocol)

### 3.1 设计原则

物理生产黑盒化、资产定义中心化。工厂只负责贴标和录入，定义权完全收归品牌总部。

### 3.2 三步流程

**第一步：平台发放空白授权**
- 向工厂下发空白子标签，数据库中只有 UID，无任何品牌/SKU 属性

**第二步：工厂端盲扫植入**
- 工厂每生产一个商品，扫描空白子标并贴入
- 工厂后台仅发送 UID 扫描记录，不显示任何商品信息

**第三步：品牌方母卡激活与身份赋予**
- 商品运抵品牌方总部后，总部人员用母标签扫描子标签
- 系统验证子标来源合法后，品牌方选择 SKU，执行母子纠缠，完成身份注入

### 3.3 漏洞封堵

| 风险场景 | 防御机制 |
|----------|----------|
| 工厂多造产品 | 工厂只有子标签，没有母标签无法赋予身份 |
| 新品泄密 | 工厂系统只有 UID，工人不知道产品是什么 |
| 标签被盗 | 偷走的只是空白 UID，品牌方不授权则无法激活 |

## 四、链式授权协议 (Chain-Linked Activation)

### 4.1 四级主权链

| 权力层级 | 载体 | 持有者 | 核心职能 |
|----------|------|--------|----------|
| L0 创世层 | Genesis Tag | 品牌全球总部 | 定义品牌基因，生成区域授权 |
| L1 渠道层 | Channel Tag | 区域总代/分公司 | 激活下辖门店权限，分配销售配额 |
| L2 零售层 | Store Tag | 品牌直营店/授权店 | 执行商品售出后的合法化点睛 |
| L3 资产层 | Asset Tag | 单品 | 承载最终主权，开启二级市场过户 |

### 4.2 链式激活流程

1. **生产预埋**：工厂将空白子卡缝入资产，资产处于待激活状态
2. **渠道赋能**：总部通过 Genesis Tag 感应 Channel Tag，分配合法化点数
3. **终端销售激活**：消费者付款后，柜员用 Store Tag 感应 Asset Tag，状态变为 LEGALLY_SOLD
4. **消费者确权**：消费者扫码完成所有权登记
5. **二次流转**：二手交易时通过双扫完成资产过户

---

## 五、权力分配：双重否决制

| 角色 | 责任 | 执行方式 |
|------|------|----------|
| 品牌方 | 主张权：判定产品质量、发往市场、销售对象 | 通过物理母卡发起点睛请求 |
| 平台方 | 仲裁权：验证母卡合法性、配额充足性、下发指令 | 通过 KMS 校验并下发加密指令 |

核心平衡：品牌方持有母卡代表行政权，子卡代表资产本身，平台持有 KMS 代表法律和公证权。

## 六、主权锁定协议 (Sovereignty Lock Protocol)

### 6.1 核心洞察：赠予权与处置权的分离

母子纠缠机制天然实现了实物资产上"赠予权与处置权的分离"——这是奢侈品消费中一个极度隐秘但刚需的痛点。

在传统奢侈品赠送逻辑中，实物一旦送出，所有权彻底失控。RC-Protocol 的母子纠缠架构从物理层面解决了这个问题，在商业史上首次实现了实物资产上"投票权与分红权的分离"。

### 6.2 权力分离模型

| 角色 | 持有物 | 权力 | 类比 |
|------|--------|------|------|
| 赠予者（A） | 母标签 Key（匙） | 资产处置权：过户、转让、二级市场流通授权 | 投票权 |
| 使用者（B） | 子标签 Crown（冠）+ 实物 | 资产使用权：穿戴、展示、享受社交红利 | 分红权 |

### 6.3 主权锁定机制

1. **赠予场景**：A 购买奢侈品后完成母子纠缠，将实物（含子标签）赠予 B，但母标签留在自己手中
2. **使用权完整**：B 可以正常使用实物，扫码验证显示"正品认证"，享受完整的社交价值
3. **处置权锁定**：B 若试图将资产在二级市场变现，系统检测到缺少母卡授权，资产处于"主权锁定"状态
4. **二级市场封杀**：任何正规买家扫码时看到"此资产主权已锁定，未获母卡授权，禁止过户"，无母卡授权的资产在二级市场视为"黑户"，价格暴跌甚至无人敢收

### 6.4 状态机扩展

| 锁定状态 | 含义 | 扫码显示 |
|----------|------|----------|
| SOVEREIGNTY_LOCKED | 母卡持有人未授权流通 | "Asset Sovereignty Locked — 主权之匙未授权此资产流通" |
| SOVEREIGNTY_RELEASED | 母卡持有人已授权流通 | 正常过户流程 |

### 6.5 主权代理模式

- **优雅托管**：母卡持有人可表达"这张主权卡我先替你放在私人金库里托管，需要保养或换新款时再找我"——这是基于物理密码学的情感契约
- **主动释放**：母卡持有人随时可通过感应母卡 + 生物识别，将资产主权释放给使用者
- **继承转移**：母卡持有人可将母卡主权迁移至使用者名下，完成完整的所有权转让

### 6.6 三大阶层痛点解决

| 痛点 | 传统困境 | RC-Protocol 方案 |
|------|----------|------------------|
| 资产定向留存 | 赠品被变现，资产非理性流失 | 母子标签确保礼物用于"穿戴"而非"套现"——给你的，才是你的；我不给，你不能抢 |
| 优雅情感约束 | 需要法律协议约束，尴尬且伤感情 | 基于物理硬件的情感契约，无需明说"你不准卖" |
| 拦截虚假繁荣 | 名媛通过变卖赠品维持社交开销 | 物理斩断"空手套白狼"链路，品牌产品真正留在目标客户手中 |

> **设计哲学**：这不是法律协议，这是基于物理密码学的情感契约。它把"人性的占有欲"和"加密算法"优雅地结合在一起。

## 七、用户体系

### 6.1 账户模型："1 + N" 模式

- **唯一 ID (UID)**：每个用户一个唯一的 V5 系统 ID
- **双重身份属性**：个人模式（收藏、炫耀、流转）与主体模式（品牌方、经销商、二奢店）
- **数字签名**：流转确权本质是原持有人"所有权放弃签名" + 新持有人"接收确认签名"

### 6.2 用户分级

| 级别 | 描述 | 权限 |
|------|------|------|
| 游客 | 首次扫码，静默登录 | 只能看真伪 |
| 普通用户 | 已注册 | 查看资产详情 |
| 高级用户 | 实名认证 | 过户、金融抵押、进入私人藏品馆 |

### 6.3 权力继承树

系统只需检查执行操作的 ID/卡最终能否追溯到品牌总部，即可判定操作合法性。

### 6.4 三因子验证逻辑

| 场景 | 结果 |
|------|------|
| 仅扫子卡 | 只读公示信息，无法编辑 |
| 仅扫母卡（无子卡在场） | 管理控制台空界面：未检测到可操作资产 |
| 登录非持有者账号 + 扫母卡 | 报错：无权管理 |
| 登录持有者账号 + 扫母卡 + 扫子卡 | 进入完整管理页面 |

### 6.5 防胁迫机制

输入特定副密码后，表面显示修改成功，后台立即向品牌方和警方发送资产主权受到非法胁迫预警，锁定该资产的二级市场交易。

## 八、资产流转（数字化过户）

1. A 在藏馆点击"发起转让"，生成临时交易码
2. B 扫码并支付确权费
3. A 确认转让（生物识别）
4. KMS 更新 Current_Owner_ID：A 到 B
5. A 的藏馆中资产消失，出现在 B 的藏馆中

## 九、防泄露与超发机制

### 8.1 所有权与操作权分离

- **工厂端设备**：仅具备盲扫权限，使用临时性 Transport Key，无法接触核心密钥
- **品牌方点睛设备**：能发起 AES 认证，修改芯片密钥并启用 SUN 动态镜像，需双重认证

### 8.2 配额控制

- 品牌方每年的点睛额度需向平台购买
- 每点睛一个商品，云端点数减 1，点数耗尽则设备立刻失效

### 8.3 物理与数字双重审计

- **芯片白名单**：只有白名单内的芯片才允许被点睛
- **异常监控**：同一 UID 短时间内被多次尝试点睛，或点睛地点异常，系统自动锁死账号

## 十、深水区补丁

### 9.1 司法级证据链

每次点睛和过户同步推送摘要至去中心化存证链，支持法庭举证。

### 9.2 资产死亡协议

- **资产报废**：母卡感应子卡执行 SET_DESTRUCTED，KMS 永久注销 UID
- **品牌方破产**：协议具备托管模式，已确权用户仍可查询真伪

### 9.3 跨国贸易清算

- 底层以 SDR 或稳定币计价
- H5 根据用户 IP 自动切换币种
- 支持全球分账的支付网关
