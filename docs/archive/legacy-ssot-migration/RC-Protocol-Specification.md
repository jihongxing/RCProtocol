## 一、项目概述

### 1.1 项目定位

RC-Protocol（Regalis Clavis Protocol）是一套基于 NFC 加密芯片（NTAG 424 DNA）的**全球实物资产数字主权协议**。
通过独创的"母子标签纠缠"机制，为高价值实物资产建立不可伪造的数字身份，
实现真伪核验、所有权确权、二手流转清算的完整闭环。

**终极形态**：实物资产的数字清算中心 — 一个中立的、基于物理密码学的"实物资产交易所"。

### 1.3 核心价值主张

## 三、核心协议规范

### 3.1 母子纠缠协议 (The Entanglement Protocol)

**激活三要素**：`[母卡权限] + [子卡芯片] + [云端指令] = 资产确权`

**纠缠流程**：

1. **物理纠缠**：点睛人员先感应母卡（持有权证明），再感应子卡（资产本体）
2. **数字加签**：App 将母卡签名信息与子卡 UID 结合发送给 KMS
3. **云端闭环**：KMS 校验母卡合法性与配额充足性，下发加密指令完成绑定

### 3.3 盲扫协议 (Blind-Entry Protocol)

**设计原则**：物理生产黑盒化、资产定义中心化

| 步骤 | 执行方 | 操作 | 信息可见性 | 状态转换 |
|------|--------|------|-----------|----------|
| 1. 发放空白授权 | 平台 | 向工厂下发空白子标签 | 数据库仅有 UID，无品牌/SKU | PRE_MINTED(0) |
| 2. 盲扫植入 | 工厂 | 扫描空白子标并贴入产品 | 工厂仅看到"扫描成功"，无商品信息 | → FACTORY_LOGGED(1) → UNASSIGNED(10) |
| 3. 母卡激活 | 品牌方 | 母标签扫描子标签，选择 SKU，执行纠缠 | 品牌方完成身份注入 | → ROTATING_KEYS(11) → ENTANGLED_PENDING(12) → ACTIVATED(2) |

**漏洞封堵**：

| 风险 | 防御 |
|------|------|
| 工厂多造 | 无母标签无法赋予身份 |
| 新品泄密 | 工厂系统仅有 UID |
| 标签被盗 | 空白 UID 无法激活 |

### 3.4 链式授权协议 (Chain-Linked Activation)

权力由上至下物理传递，每一级必须获得上一级的"物理感应 + 云端授权"：

1. **生产预埋**：工厂将空白子卡缝入资产 → 待激活
2. **渠道赋能**：Genesis Tag 感应 Channel Tag → 分配合法化点数
3. **终端销售**：Store Tag 感应 Asset Tag → LEGALLY_SOLD
4. **消费者确权**：扫码完成所有权登记
5. **二次流转**：双扫完成资产过户

### 3.5 主权锁定协议 (Sovereignty Lock Protocol)

**核心洞察**：母子纠缠机制天然实现了实物资产上"赠予权与处置权的分离"——这是奢侈品消费中一个极度隐秘但刚需的痛点。

在传统奢侈品赠送逻辑中，实物一旦送出，所有权彻底失控。RC-Protocol 的母子纠缠架构从物理层面解决了这个问题：

**权力分离模型**：

| 角色 | 持有物 | 权力 | 类比 |
|------|--------|------|------|
| 赠予者（A） | 母标签 Key（匙） | 资产处置权：过户、转让、二级市场流通授权 | 投票权 |
| 使用者（B） | 子标签 Crown（冠）+ 实物 | 资产使用权：穿戴、展示、享受社交红利 | 分红权 |

**主权锁定机制**：

1. **赠予场景**：A 购买奢侈品后完成母子纠缠，将实物（含子标签）赠予 B，但母标签留在自己手中
2. **使用权完整**：B 可以正常使用实物，扫码验证显示"正品认证"，享受完整的社交价值
3. **处置权锁定**：B 若试图将资产在二级市场变现，系统检测到缺少母卡授权，资产处于"主权锁定"状态
4. **二级市场封杀**：任何正规买家扫码时看到"此资产主权已锁定，未获母卡授权，禁止过户"，无母卡授权的资产在二级市场视为"黑户"

**状态机扩展**：

| 锁定状态 | 含义 | 扫码显示 |
|----------|------|----------|
| SOVEREIGNTY_LOCKED | 母卡持有人未授权流通 | "Asset Sovereignty Locked — 主权之匙未授权此资产流通" |
| SOVEREIGNTY_RELEASED | 母卡持有人已授权流通 | 正常过户流程 |

**主权代理模式**：

- **托管话术**：母卡持有人可优雅地表达"这张主权卡我先替你放在私人金库里托管，需要保养或换新款时再找我"
- **主动释放**：母卡持有人随时可通过感应母卡 + 生物识别，将资产主权释放给使用者
- **继承转移**：母卡持有人可将母卡主权迁移至使用者名下，完成完整的所有权转让

**商业价值**：

| 维度 | 价值 |
|------|------|
| 双端锁定 | 母卡持有人 = 终身忠实用户（处置权），使用者 = 日常高频用户（验证权），一次赠予锁定两个高净值终端 |
| 品牌方利益 | 确保产品真正留在目标客户手中，而非流向混乱的批发市场，维护品牌价值 |
| 特权服务费 | 主权锁定/释放/代理托管可收取高端特权服务费 |
| 供应链净化 | 物理斩断"赠品变现"链路，拦截通过变卖赠品维持社交开销的虚假繁荣 |

> **设计哲学**：给你的，才是你的；我不给，你不能抢。这不是法律协议，这是基于物理密码学的情感契约。

## 四、安全体系

### 4.1 KMS 密钥管理系统

#### 分层衍生体系 (KDF)

```
Root_Key (HSM 物理隔离)
  └── Brand_Key = HMAC-SHA256(Root_Key, Brand_ID ‖ System_ID)   // 品牌派生密钥
        ├── K_chip = HMAC-SHA256(Brand_Key, UID ‖ Epoch)[..16]  // 芯片终端密钥 (AES-128)
        └── K_honey = HMAC-SHA256(Brand_Key, Serial_BE)          // 蜜标密钥 (256-bit HMAC key)
```

- `System_ID`：地理/逻辑隔离盐值，防止跨系统密钥碰撞
- `Epoch`：密钥轮换版本号（u32 LE），预留 Key Rotation 能力
- `Serial_BE`：蜜标序列号（u32 BE），用于 Honey Badger 标记系统
- 数据库仅存 UID，密钥实时派生，绝不明文存储
- 所有加密运算在 TEE（可信执行环境）隔离内存中完成
- 中间密钥（Brand_Key）在派生出下级密钥后立即清零（Zeroize）

> **代码权威**：密钥派生实现以 `rc-core/src/crypto/kdf.rs` 为准。

#### 核心组件

| 组件 | 功能 |
|------|------|
| HSM 加密机 | 物理隔离存储根密钥 |
| TEE 安全执行环境 | 隔离内存中完成加密运算 |
| 动态指令生成器 | UID → 验证配额 → 生成 K2 → 封装 APDU 指令流 |
| 确权认证网关 | UID+CTR+CMAC → 派生 K2 → 对比 CMAC → 发放 JWT |

#### 熔断机制

| 触发条件 | 动作 |
|----------|------|
| CTR ≤ 历史记录 | 判定克隆卡攻击，拉黑请求 |
| 单 UID 1分钟内查询 >10 次 | 冻结验证功能 |
| 5分钟内跨洲扫码 | 触发风控标记 |
| 母卡短时间内纠缠大量子标 | 封锁该 M-UID |

### 4.2 母卡防克隆保护

**三位一体防伪架构**：

| 层级 | 机制 | 原理 |
|------|------|------|
| 物理层 | PUF 芯片 | 半导体物理差异产生唯一"硅指纹"，无法克隆 |
| 算法层 | 双向动态握手 | App 发随机数 → 母卡内部 HMAC → 密钥永不出芯片 |
| 策略层 | 滚动计数器 | 不可逆计数器，流水号回溯或跳变 → 全局熔断 |

**母卡分级管理**（物理层组织架构，密钥派生逻辑见 KDF 章节）：

| 级别 | 名称 | 持有者 | 权限 |
|------|------|--------|------|
| L1 | Genesis（创世） | CEO / 首席法务 | 激活下级母卡；重置品牌根密钥 |
| L2 | Regional（区域） | 区域负责人 | 在授权额度内分配 |
| L3 | Operator（点睛） | 一线质检员 | 执行子卡点睛 |

> L0 为离线根密钥（Root_Key），物理隔离于 HSM 中，不对应任何卡片实体。

**挂失流程**：
- 紧急挂失 (Kill Switch)：Console 点击 → KMS 黑名单 → 全球失效
- L0 丢失修复：CEO + 技术总监 + 平台方三方多签 → 生成新根密钥

### 4.3 设备端安全接入

**双层证书体系**：

| 设备级别 | 权限 | 安全要求 |
|----------|------|----------|
| L1 Factory Reader | 仅盲扫 | 任务制证书，过期自动吊销 |
| L2 Brand Activator | 纠缠与定义 | 绑定品牌方硬件安全密钥 |

**通讯安全**：
- mTLS 双向认证（设备证书 + 私钥签名 + 订单令牌）
- HMAC 校验每条 API 请求 Body
- 时钟同步（误差 ±30s），防离线重放

### 4.4 全链路加密

| 层级 | 方案 |
|------|------|
| 通讯层 | 强制 mTLS，每终端独立设备证书 |
| 数据层 | AES-256-GCM 二次加密，设备私钥签名 + 平台公钥加密 |
| 密钥派生 | `K_tag = CMAC(K_master, UID ∥ System_ID)`，单芯片破解不影响全局 |
| SUN 消息 | AES-128 动态密文 + 自增计数器防重放 |

### 4.5 隐私主权架构

**核心原则**："零知识、物理隔离、主权自持"

| 机制 | 实现 | 效果 |
|------|------|------|
| 零知识验证 | 云端仅存匿名权属令牌，生物信息留本地 | 数据库泄露无法对应自然人 |
| 阅后即焚 | 地理位置仅用于风控，验证后物理抹除 | 不记录用户轨迹 |
| 主权冷存储 | 核心权属密钥存于用户母标签硬件 | 平台倒闭不影响资产归属 |
| 最小化数据 | 匿名手机号(SHA-256加盐) + 母标签 + 生物特征 | 分布式信任，无中心化身份库 |
| 品牌方隔离 | 品牌方仅见资产流转状态，联系用户须经匿名中转 | 用户隐私绝对保护 |

### 5.3 SDM 配置 (Secure Dynamic Messaging)

**密钥分配**：

| 密钥 | 用途 |
|------|------|
| K0 (Master Key) | 修改配置 |
| K1 (SDM Meta Key) | 加密 UID（可选） |
| K2 (SDM File Read Key) | 生成动态 CMAC（核心） |

**动态 URL 结构**：

```
https://rc-protocol.com/v?u=00000000000000&c=000000&m=0000000000000000
```

| 参数 | 长度 | 说明 |
|------|------|------|
| u (UID) | 7 字节 | 芯片全球唯一序列号（明文） |
| c (CTR) | 3 字节 | NFC 计数器，每感应 +1 |
| m (CMAC) | 8 字节 | AES-128 动态密文 |

**APDU 配置流程**：

1. 使用初始密钥进行 AES 认证
2. 启用 SDM，配置 CTR/CMAC 偏移位置（FileOption: 0x40, AccessRights: 0x1211, SDMOptions: 0x81）
3. 写入带占位符的基础 URL
4. 开启 Mirror CTR
5. 修改 K0 并锁定配置区（不可逆）

### 5.4 芯片内属性码 (4字节 Sovereign Descriptor)

```
品类码(1B) + 品牌系列码(1B) + MSRP(2B)
示例：02-01-AF40 → [包类]-[香奈儿系列]-[¥45,000]
```

### 6.4 消费者端 — 两步验证协议

消费者扫码验证采用两步协议设计，将安全验证与业务数据获取解耦：

#### Step 1：SUN 消息解析与验证

```
GET /v1/resolve/{sun_msg}
```

**功能**：解析 SUN 动态消息，执行 CMAC 验证、计数器校验、地理围栏检查，返回验证状态。

**返回字段**：
- `verification_status`：验证状态（见下表）
- `verification_token`：仅 GENUINE 时签发的一次性令牌
- `security_level`：安全等级（Normal / Elevated）

**8 种验证状态**：

| 状态 | 含义 | 是否签发 Token |
|------|------|----------------|
| GENUINE | 正品认证通过 | ✅ 签发 Verification Token |
| COUNTERFEIT | 伪造品检测 — CMAC 校验失败 | ❌ |
| REPLAY_ATTACK | 重放攻击 — CTR ≤ 历史记录 | ❌ |
| RATE_LIMITED | 频率限制 — 超出速率阈值 | ❌ |
| GEO_FENCE_VIOLATION | 地理围栏违规 — 非授权区域 | ❌ |
| ASSET_DISPUTED | 资产争议冻结中 | ❌ |
| SECURITY_VERIFICATION_PENDING | 安全验证待确认 — 降级响应 | ❌ |
| UNKNOWN | 未注册资产 — UID 不存在 | ❌ |

**Verification Token 机制**：
- 签发条件：验证状态为 GENUINE 且 UID、CTR 均存在
- TTL：5 分钟
- 一次性使用：通过 JTI（JWT ID）去重，使用后立即失效
- 载荷：包含 asset_uid（脱敏）、verification_status、签发时间

**SecurityVerificationPending 降级响应**：
当资产处于 SecurityVerificationPending 状态时，返回降级响应，不包含 UID、CTR、Token 信息，`security_level` 标记为 `Elevated`。持有者需通过 Challenge/Acknowledge 流程确认安全警报后恢复正常验证。

#### Step 2：凭 Token 获取完整业务数据

```
GET /v1/asset/verified
Authorization: Bearer {verification_token}
```

**功能**：凭 Step 1 签发的 Verification Token 获取完整业务数据，包括资产详情、持有人证书、多维行情（V-Value）、品牌方内容等。

**前置条件**：
- Token 未过期（5 分钟 TTL）
- Token 未被使用过（JTI 一次性校验）
- Token 验证状态为 GENUINE

**返回内容**：
- 资产详情（品牌、SKU、状态）
- 持有人证书信息
- V-Value 多维行情数据（MSRP / 市场中位数 / 回购参考价）
- 品牌方自定义内容

### 6.5 后端校验逻辑

```python
def verify_tag(uid, ctr, cmac_from_url):
    k2 = derive_key(uid)                          # 1. 派生密钥
    if int(ctr) <= last_recorded_ctr(uid):         # 2. 计数器校验
        return "REPLAY_ATTACK_DETECTED"
    sv = construct_sdm_vector(uid, ctr)            # 3. 构造计算素材
    expected_cmac = aes_cmac(k2, sv)               # 4. AES-128 CMAC
    if cmac_from_url == expected_cmac:              # 5. 比对
        update_last_ctr(uid, ctr)
        return "SUCCESS_GENUINE_ASSET"
    return "COUNTERFEIT_DETECTED"
```

### 6.6 安全审计规则

| 规则 | 触发条件 | 动作 |
|------|----------|------|
| Anti-Replay | CTR ≤ 历史值 | 判定克隆攻击 |
| Geo-Fencing | 非授权经纬度 | 标记 SUSPICIOUS |
| Entropy Check | 母卡短时间大量纠缠 | 封锁 M-UID |

| 母子纠缠 (Entanglement) | 母标签与子标签通过加密算法建立的不可伪造的物理绑定关系 |
| 点睛 (Activation) | 品牌方使用母卡触碰子卡，赋予资产数字身份的操作 |
| 盲扫 (Blind-Entry) | 工厂端在不知道商品信息的情况下扫描并植入空白子标签 |
| 子弹 (Quota) | 品牌方向平台购买的点睛配额 |
| Crown（冠） | 子标签代号，附着于实物资产的身份凭证 |
| Key（匙） | 母标签代号，掌控主权的根密钥载体 |
| SUN | Secure Unique NFC，NTAG 424 DNA 动态消息认证 |
| SDM | Secure Dynamic Messaging，安全动态消息 |
| CMAC | 基于 AES-128 的消息认证码 |
| KMS | 密钥管理系统 |
| HSM | 硬件安全模块 |
| TEE | 可信执行环境 |
| PUF | 物理不可克隆函数 |
| Sovereign Vault | 数字私人藏品馆（C 端） |
| Sovereign Console | 管理后台（B 端） |
| Sovereign Fee | 动态确权认证费 |
| V-Value | 多维价值看板 |
| Honor Display Room | 荣誉陈列室 |
| Consumed | 资产终态：物理使用殆尽，数字主权转入荣誉态 |
| Legacy | 资产终态：物理损坏不可修复，残缺美永久保留 |
| Consumption Tiering | 消费累积分级体系 |
| Sovereignty Lock | 主权锁定机制，赠予者保留母卡（处置权），使用者持有实物+子标签（使用权），实现赠予权与处置权的物理分离 |
| SOVEREIGNTY_LOCKED | 资产主权锁定状态，母卡持有人未授权流通，二级市场视为"黑户" |
| SOVEREIGNTY_RELEASED | 资产主权释放状态，母卡持有人已授权流通 |
| Kill Switch | 紧急挂失机制 |
| APDU | Application Protocol Data Unit，智能卡通讯指令 |
