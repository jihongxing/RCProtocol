## 2. 核心对象

### 2.1 Asset

被 RCProtocol 管理的实物资产。它是状态机、权属流转、安全校验、估值与审计的中心对象。

### 2.2 Tag

附着于资产上的物理标识与认证载体。当前实现基线为 `NTAG 424 DNA`。Tag 提供 UID、计数器、动态认证消息等能力。

### 2.3 Mother Tag / Authority Device

用于发起授权、激活、处置或绑定确认的物理或逻辑授权载体。

在历史文档中出现过：

- 母卡
- 母标签
- Key
- Authority Device

在当前文档体系中统一理解为：**授权绑定的上级授权载体**。

如果语境强调历史母子设计，可称 `Mother Tag`；如果语境强调更一般化的治理能力，可称 `Authority Device`。

### 2.4 Child Tag / Asset Tag

附着于实物上的资产标签。历史上也称：

- 子标签
- Crown
- Asset Tag

当前统一理解为：**资产本体所依附的物理身份载体**。

### 2.5 Entanglement / Authorization Binding

历史文档中常称“母子纠缠”。当前统一定义为：

> 上级授权载体与资产标签之间建立的、受协议约束的授权绑定关系。

该关系用于：

- 激活资产
- 约束处置权
- 支撑近场或多因子交互
- 建立资产主权前提

“纠缠”保留为业务叙事词；“授权绑定”是工程与规范用词。

### 2.6 Blind Scan / Blind Entry

工厂端或生产端只登记 UID 与最小必要生产记录，不接触品牌定义与商品详情的流程。

### 2.7 Activation

将待分配资产转化为协议内已激活资产的过程，通常涉及：

- 密钥翻转
- 绑定建立
- 品牌认领
- 状态推进

### 2.8 Ownership

资产在协议中的当前合法持有关系。Ownership 不是单纯的“谁拿着实物”，而是协议确认的合法持有状态。

### 2.9 Transfer

协议内的权属变更流程。它要求状态、权限、校验条件全部满足，才可完成所有权更新。

### 2.10 Dispute

资产因风险、争议、异常认证、审计问题等原因进入冻结审查状态的治理过程。

## 3. 角色对象

### 3.1 Platform

平台方。负责：

- 系统治理
- 密钥托管边界
- 审计
- 管理员覆盖操作
- 平台级风控与恢复

### 3.2 Brand

品牌方。负责：

- 资产定义
- 激活
- 产品信息管理
- 销售合法化流程

### 3.3 Factory

工厂或生产方。负责：

- 盲扫登记
- 生产入库阶段的最小记录

### 3.4 Consumer

消费者或合法持有者。负责：

- 验真
- 过户参与
- 消耗 / 传承类终态操作

### 3.5 Moderator

审核员 / 安全审核角色。负责：

- 冻结
- 恢复
- 篡改与失陷标记
- 争议处理

## 4. 技术对象

### 4.1 Root Key

系统根密钥。位于最高安全边界，用于派生品牌密钥。

### 4.2 Brand Key

基于 `Root Key + Brand ID + System ID` 派生出的品牌级密钥。

### 4.3 Chip Key / K_chip

基于品牌密钥与标签 UID、epoch 等信息派生的芯片终端密钥，用于动态认证。

### 4.4 K_honey

用于 HBM 或蜜标相关校验的 HMAC 密钥。

### 4.5 CTR

标签或认证消息中使用的计数器，用于防重放与异常检测。

### 4.6 CMAC / HMAC

RCProtocol 当前核心认证与派生中使用的两类密码学机制：

- `HMAC-SHA256`：用于派生
- `AES-128 CMAC`：用于动态认证消息验证

## 5. 产品对象

### 5.1 Sovereign Vault

C 端资产馆 / 数字资产档案馆，用于展示持有资产、验证结果、状态与流转记录。

### 5.2 Sovereign Preview

用于在受控场景下分享资产预览、展示权属信息或引导交易前校验的受限预览能力。

### 5.3 Sovereign Console / BConsole

B 端治理后台，用于品牌、平台、工厂和运营角色执行审批、管理、发布、审计和治理动作。

## 7. 文档术语清理结论

本次整理后，以下处理成立：

- “母子纠缠”保留为业务表述，但规范术语采用“授权绑定”
- “主权”是业务层总称，不替代状态、权限、密钥、所有权这些精确定义
- “协议宪法”“主权哲学”等表述不再作为工程术语来源
- 所有核心对象一律以本文件定义为准
