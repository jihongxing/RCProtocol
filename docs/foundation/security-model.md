# 安全模型

> 文档类型：Foundation  
> 状态：Active  
> 权威级别：Authoritative

---

## 1. 权威说明

本文件定义 RCProtocol 的安全边界、密钥体系、动态认证、冻结原则与实现约束。

凡涉及密钥、认证、风控与安全状态的问题，均以本文件为准。

---

## 2. 安全目标

RCProtocol 的安全体系服务于以下目标：

1. 验证物理标签真实性
2. 防止重放、克隆与伪造
3. 保证品牌之间密钥隔离
4. 保证平台治理与品牌发行边界可区分
5. 支撑冻结、恢复与审计

---

## 3. 密钥派生基线

当前采用三级派生链：

```text
Root_Key
  -> Brand_Key = HMAC-SHA256(Root_Key, Brand_ID || System_ID)
  -> K_chip = HMAC-SHA256(Brand_Key, UID || Epoch_LE)[..16]
  -> K_honey = HMAC-SHA256(Brand_Key, Serial_BE)
```

### 3.1 Root Key

系统根密钥，位于最高安全边界。它不参与业务层直接暴露。

### 3.2 Brand Key

品牌级派生密钥，用于构建品牌隔离的密钥空间。

### 3.3 K_chip

芯片终端密钥，当前作为 `AES-128` 认证与动态校验的核心密钥材料。

### 3.4 K_honey

用于蜜标 / HBM 等扩展校验的 HMAC 密钥，不做 16 字节截断。

---

## 4. 动态认证基线

### 4.1 SUN / 动态消息

当前基线采用以动态消息为核心的标签认证方式。

最小验证元素包括：

- UID
- CTR
- CMAC

### 4.2 Mode A

MVP 阶段可使用明文兼容模式，在 URL 中传递 UID 与 CTR，并对动态消息进行 CMAC 校验。

### 4.3 Mode B

支持加密隐私模式，将 UID / CTR 通过 SDM 加密传输，用于更高隐私等级场景。

---

## 5. 风险检测规则

### 5.1 计数器回滚

如果接收到的 CTR 小于或等于历史记录，应视为高风险事件，通常意味着重放或克隆尝试。

### 5.2 高频访问

同一资产在异常时间窗口内被高频读取，应触发限流、降级或冻结前检查。

### 5.3 地理围栏

短时间内出现不合理的跨区域认证行为，应触发风控标记或冻结流程。

### 5.4 纠缠异常 / 绑定异常

当授权绑定关系与期望关系不匹配时，应拒绝后续处置动作。

---

## 6. 冻结与终态原则

### 6.1 Disputed

`Disputed` 是冻结治理状态，不等于安全终态。它用于：

- 审核争议
- 等待恢复
- 隔离风险资产

### 6.2 Tampered

`Tampered` 用于确认物理篡改或物理安全链路断裂后的终态标记。

### 6.3 Compromised

`Compromised` 用于确认已泄露、已失陷或无法恢复信任的终态标记。

---

## 7. 授权绑定与近场约束

历史文档大量使用“母子纠缠”表述。当前整理后确认：

- 它是当前系统的重要业务与安全能力
- 但其正式工程含义统一归入“授权绑定”
- 若绑定关系用于处置、过户、释放等动作，则必须满足相应验证前提

是否要求近场双因子、双标签同时在场、或更复杂的绑定校验，由具体产品流程决定，但其安全判断必须服从本文件与 API / 服务边界定义。

---

## 8. 安全实现约束

### 8.1 密钥对象要求

- 实现 `Zeroize` / `ZeroizeOnDrop`
- 禁止明文泄漏到日志
- 禁止普通比较方式处理敏感值
- 禁止无约束复制敏感密钥材料

### 8.2 比较要求

密码学值比较必须采用常量时间比较。

### 8.3 中间密钥处理

派生链中的中间密钥应尽快清零，避免在非必要范围内驻留内存。

### 8.4 审计要求

所有安全相关动作必须可追踪，包括：

- 冻结
- 恢复
- 认证失败
- 密钥轮换过程
- 高风险策略触发

---

## 9. 当前明确废止的安全叙事写法

以下内容可作为历史思路保留，但不再作为当前安全规范来源：

- 四级密钥派生链直接等同于代码实现
- 未落地的离线纠缠公式
- 尚未稳定实现的影子参数扩展
- 纯愿景式“绝对不可克隆”描述

当前规范只保留：

- 已明确的密钥派生链
- 已明确的动态认证规则
- 已明确的风险与冻结原则
