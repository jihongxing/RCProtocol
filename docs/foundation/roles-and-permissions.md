# 角色与权限

> 文档类型：Foundation  
> 状态：Active  
> 权威级别：Authoritative

---

## 1. 权威说明

本文件定义 RCProtocol 的角色模型、权限边界与状态转换权限来源。

凡涉及“谁可以做什么”的问题，均以本文件为准。

---

## 2. 角色列表

### 2.1 Platform

平台方角色，具备系统治理、全局审计、管理员覆盖与恢复能力。

### 2.2 Factory

工厂 / 生产方角色，仅负责盲扫登记、入库链路中的有限操作。

### 2.3 Brand

品牌方角色，负责资产定义、激活、销售合法化、产品信息维护。

### 2.4 Consumer

消费者 / 合法持有者角色，负责验真、过户参与、消费 / 传承类动作。

### 2.5 Moderator

审核员 / 安全审核角色，负责冻结、恢复、安全标记与争议处置。

---

## 3. 权限原则

### 3.1 最终拒绝权原则

上层治理系统可以放行请求，但最终是否允许执行，必须由协议核心根据角色、状态和业务约束再次判定。

### 3.2 角色权限与状态耦合

权限不是静态按钮权限，而是：

- 当前角色
- 当前资产状态
- 当前租户 / 品牌边界
- 当前审批 / 审计上下文

共同决定。

### 3.3 平台覆盖不等于无审计

`Platform` 可以执行管理员级合法转换，但所有覆盖动作必须可追踪、可审计、可解释。

---

## 4. 状态转换权限矩阵

| 角色 | 允许的核心转换 |
|------|----------------|
| `Platform` | 任意合法转换 |
| `Factory` | `PreMinted -> FactoryLogged`，`FactoryLogged -> Unassigned` |
| `Brand` | `Unassigned -> RotatingKeys -> EntangledPending -> Activated -> LegallySold` |
| `Consumer` | `LegallySold -> Transferred | Consumed | Legacy`，`Transferred -> Transferred | Consumed | Legacy` |
| `Moderator` | `any -> Tampered | Compromised | Disputed`，`Disputed -> previous_state` |

说明：

- 这里的 `any` 指任意非终态或允许治理接管的上下文
- `previous_state` 表示恢复到冻结前的合法状态

---

## 5. 角色职责边界

### 5.1 Platform

允许：

- 全局治理
- 管理员覆盖
- 审计与恢复
- 平台级参数与策略管理

不应承担：

- 代替品牌定义商品内容
- 伪造品牌激活动作的业务语义

### 5.2 Factory

允许：

- 盲扫登记
- 入库记录
- 最小生产链路反馈

禁止：

- 访问品牌定义内容
- 激活资产
- 修改资产持有关系

### 5.3 Brand

允许：

- 认领资产
- 激活资产
- 完成销售合法化
- 维护商品内容与品牌侧数据

禁止：

- 绕过平台规则直接修改系统安全结论
- 在终态资产上执行非法回滚

### 5.4 Consumer

允许：

- 验真
- 发起 / 参与转让
- 将资产推进到消费或传承终态

禁止：

- 修改品牌信息
- 修改平台安全判定
- 绕过授权绑定直接过户

### 5.5 Moderator

允许：

- 冻结争议资产
- 标记篡改 / 失陷
- 执行恢复流程

禁止：

- 直接行使品牌经营职责
- 用安全角色替代业务角色完成正常销售或转让

---

## 6. 权限执行层次

权限检查分为两层：

### 6.1 外层治理权限

由治理系统负责：

- 组织与岗位映射
- RBAC / ABAC
- 审批流
- 策略版本
- 操作上下文

### 6.2 内层协议权限

由核心协议层负责：

- 角色是否合法
- 状态转换是否合法
- 租户 / 品牌边界是否合法
- 当前资源是否允许该动作

协议层拥有最终裁决权。

---

## 7. 与审批、幂等、审计的关系

凡涉及治理写操作，应同时满足：

- 请求可鉴权
- 写操作可幂等
- 关键动作有审批上下文
- 审计事件完整记录

如果这些条件不满足，即便角色本身有权限，也不应视为可执行。

---

## 8. 清理结论

本次整理后：

- 角色体系统一为 5 角色模型
- 所有“总部 / 渠道 / 门店 / 创世卡”等层级，不再作为平台角色枚举，而是业务或硬件分层概念
- 权限矩阵与状态机强绑定，不再允许在其他文档中重复定义不同版本
