### 7. ✅ Webhook 推送服务

**位置：** `services/go-webhook/`

**架构：**
- 独立的 Go 服务
- 基于数据库队列的异步推送
- 多 Worker 并发处理
- 自动重试机制（3 次，指数退避）
- HMAC-SHA256 签名验证
- 超时控制（30 秒）

**核心模块：**
- `cmd/webhook/main.go` - 服务入口
- `internal/config/config.go` - 配置管理
- `internal/queue/db.go` - 数据库操作
- `internal/queue/queue.go` - 队列管理
- `internal/delivery/worker.go` - 推送 Worker

**支持的事件类型：**
- `asset.activated` - 资产激活
- `asset.sold` - 资产售出
- `asset.transferred` - 所有权转移
- `asset.disputed` - 资产冻结
- `asset.recovered` - 资产恢复

**重试策略：**
- 第 1 次失败：1 分钟后重试
- 第 2 次失败：5 分钟后重试
- 第 3 次失败：15 分钟后重试
- 超过 3 次：标记为 Abandoned

## 数据库访问

```bash
# 进入 PostgreSQL 容器
docker exec -it rcprotocol-postgres-1 psql -U rcprotocol -d rcprotocol

# 查看品牌
SELECT brand_id, brand_name, status FROM brands;

# 查看 API Keys
SELECT key_id, brand_id, key_prefix, status FROM api_keys;

# 查看资产
SELECT asset_id, uid, brand_id, current_state, external_product_id FROM assets;

# 查看状态事件
SELECT event_id, asset_id, action, from_state, to_state, actor_role, occurred_at 
FROM asset_state_events 
ORDER BY occurred_at DESC 
LIMIT 10;

# 查看 Webhook 推送
SELECT delivery_id, event_type, status, attempts, created_at 
FROM webhook_deliveries 
ORDER BY created_at DESC 
LIMIT 10;
```
