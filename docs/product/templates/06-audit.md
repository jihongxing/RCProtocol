# Audit 模块设计模板实例

> 文档类型：Product  
> 状态：Active  
> 权威级别：Execution Template

---

## 1. 模块信息

- 模块名称：Audit
- 当前阶段：MVP
- 对应页面：审计查询页、审计详情抽屉 / 详情页
- 对应角色：Platform、Moderator、Brand

---

## 2. 一句话商业价值

让平台、品牌和审核员能够查询关键操作事实，支撑交付、争议处理与续费信任，是后台治理可落地的最低保障。

---

## 3. 模块目标

### 3.1 要解决的问题

- 关键写操作若无法查询，品牌无法信任治理后台
- 出现争议时，没有审计事实就无法追责和恢复
- 平台需要最小可用的查询入口支撑试点运维

### 3.2 本模块必须达成的结果

- Platform 能检索关键审计记录
- Brand 能查看与本租户相关的审计摘要
- Moderator 能查看争议与恢复相关记录

### 3.3 本模块边界

本模块负责：

- 审计检索
- 审计详情查看
- 基础筛选与追踪

本模块不负责：

- 高级 BI 报表
- 自动风险决策引擎
- 复杂可视化分析系统

---

## 4. 目标角色与使用场景

| 角色 | 是否主角色 | 使用场景 | 核心诉求 |
|------|------------|----------|----------|
| Platform | 是 | 查询全局审计事实 | 全局可见、可追责 |
| Brand | 是 | 查询本品牌关键动作 | 能解释业务链路 |
| Factory | 否 | 一般不使用 | 无 |
| Consumer | 否 | 不使用 | 无 |
| Moderator | 是 | 查询争议 / 恢复记录 | 事实完整 |

---

## 5. 最小用户路径

1. 用户进入审计查询页
2. 选择条件进行检索
3. 查看列表结果
4. 打开详情查看状态变化与上下文

### 5.2 页面跳转 / 任务推进

| 步骤 | 页面 / 容器 | 用户动作 | 系统反馈 | 结果 |
|------|-------------|----------|----------|------|
| 1 | 审计查询页 | 输入筛选条件 | 返回结果列表 | 可浏览记录 |
| 2 | 结果列表 | 打开某条记录 | 展示详情 | 查看上下文 |
| 3 | 详情页 / 抽屉 | 查看变化链路 | 显示字段与状态 | 支撑判断 |

## 7. 状态机映射

### 7.1 涉及状态

本模块本身不推进状态，但必须能够展示各关键写操作的状态变化事实。

### 7.2 页面动作与状态变化映射

| 页面动作 | 操作角色 | 前置状态 | 结果状态 | 是否正式状态变化 |
|----------|----------|----------|----------|------------------|
| 查询审计记录 | Platform / Brand / Moderator | 无 | 无 | 否 |
| 查看审计详情 | Platform / Brand / Moderator | 无 | 无 | 否 |

---

| 动作 / 页面 | Platform | Brand | Factory | Consumer | Moderator |
|-------------|----------|-------|---------|----------|-----------|
| 访问审计页 | 是 | 是 | 否 | 否 | 是 |
| 查看全局记录 | 是 | 否 | 否 | 否 | 受限 |
| 查看本品牌记录 | 是 | 是 | 否 | 否 | 受限 |
| 查看争议 / 恢复记录 | 是 | 受限 | 否 | 否 | 是 |

## 9. 接口动作映射

---

## 10. 审计与追踪要求

本模块读取审计数据，但其自身的查询行为也应记录最小访问日志，至少包含查询者、时间、筛选条件摘要。

---

## 11. 异常流

### 异常流 1

- 触发条件：Brand 访问超出租户边界的记录
- 页面表现：返回无权限
- 用户下一步：缩小范围或联系平台
- 是否需要人工兜底：否

### 异常流 2

- 触发条件：查询条件过宽导致结果为空或超时
- 页面表现：提示优化筛选条件
- 用户下一步：调整筛选项
- 是否需要人工兜底：否

### 异常流 3

- 触发条件：详情记录不存在或已清理
- 页面表现：显示记录不可用
- 用户下一步：返回列表
- 是否需要人工兜底：可选

---

## 12. 不做什么

- 不做复杂图表分析中心
- 不做高级风控工作台
- 不做跨维度数据仓库能力

---

## 13. 验收标准（DoD）

### 13.1 产品验收

- Platform 能检索到关键审计事件
- Brand 能查看本品牌相关记录
- Moderator 能查看争议相关记录

### 13.2 联调验收

- 激活、售出、盲扫等关键写操作均可被检索
- 记录详情包含核心上下文字段
- 权限边界正确生效

### 13.3 演示验收

- 可现场查询某资产从盲扫到售出的关键链路记录

---

## 14. 人工兜底策略

| 场景 | 是否允许人工兜底 | 兜底方式 | 上线前是否必须系统化 |
|------|------------------|----------|----------------------|
| 复杂报表分析 | 是 | 导出后人工分析 | 否 |
| 特殊争议复盘 | 是 | 平台人工整理 | 否 |
