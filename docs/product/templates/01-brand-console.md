# Brand Console 模块设计模板实例

> 文档类型：Product  
> 状态：Active  
> 权威级别：Execution Template

---

## 1. 模块信息

- 模块名称：Brand Console
- 当前阶段：MVP
- 对应页面：品牌管理页、产品管理页、品牌详情页、产品详情页
- 对应角色：Brand、Platform

---

## 2. 一句话商业价值

让品牌方能独立完成品牌 / 产品录入与维护，形成接入起点，否则 RCProtocol 无法进入真实试点交付。

---

## 3. 模块目标

### 3.1 要解决的问题

- 品牌试点接入缺少统一录入入口
- 产品定义分散会导致后续激活、售出、验真链路失真
- 平台与品牌之间缺少可审计的主数据入口

### 3.2 本模块必须达成的结果

- Brand 可创建并维护品牌资料
- Brand 可创建并维护产品资料
- Platform 可查看并辅助治理品牌主数据

### 3.3 本模块边界

本模块负责：

- 品牌基础资料录入
- 产品基础资料录入
- 品牌 / 产品列表与详情查询

本模块不负责：

- 复杂审批流系统
- 多品牌策略中心
- 商品营销内容 CMS

---

## 4. 目标角色与使用场景

| 角色 | 是否主角色 | 使用场景 | 核心诉求 |
|------|------------|----------|----------|
| Platform | 是 | 查看、辅助接入、做全局治理 | 能看到品牌接入进度 |
| Brand | 是 | 创建品牌与产品 | 快速完成试点录入 |
| Factory | 否 | 只读取必要产品信息 | 不承担品牌主数据维护 |
| Consumer | 否 | 不使用 | 无 |
| Moderator | 否 | 不使用 | 无 |

---

## 5. 最小用户路径

1. 品牌管理员登录后台
2. 创建或补全品牌资料
3. 创建试点产品
4. 保存后进入后续批次 / 激活流程

### 5.2 页面跳转 / 任务推进

| 步骤 | 页面 / 容器 | 用户动作 | 系统反馈 | 结果 |
|------|-------------|----------|----------|------|
| 1 | 品牌列表页 | 新建品牌 | 表单校验成功 | 生成品牌记录 |
| 2 | 品牌详情页 | 补全品牌字段 | 保存成功提示 | 品牌可用 |
| 3 | 产品列表页 | 新建产品 | 表单校验成功 | 生成产品记录 |
| 4 | 产品详情页 | 确认产品可投产 | 状态提示完成 | 可进入 Factory Flow |

## 6. 页面结构

### 6.1 页面清单

| 页面 | 目标 | 主操作 | 次操作 | 备注 |
|------|------|--------|--------|------|
| 品牌列表页 | 查看已有品牌 | 新建品牌 | 搜索 / 筛选 | MVP 保持简洁 |

## 7. 状态机映射

### 7.1 涉及状态

本模块不直接推进资产状态机，但它为后续 `Unassigned -> Activated -> LegallySold` 链路提供品牌与产品上下文。

### 7.2 页面动作与状态变化映射

| 页面动作 | 操作角色 | 前置状态 | 结果状态 | 是否正式状态变化 |
|----------|----------|----------|----------|------------------|
| 新建品牌 | Brand / Platform | 无 | 无 | 否 |
| 新建产品 | Brand / Platform | 无 | 无 | 否 |

---

## 8. 权限与可见性

| 动作 / 页面 | Platform | Brand | Factory | Consumer | Moderator |
|-------------|----------|-------|---------|----------|-----------|
| 页面访问 | 是 | 是 | 否 | 否 | 否 |
| 创建品牌 | 是 | 是 | 否 | 否 | 否 |
| 创建产品 | 是 | 是 | 否 | 否 | 否 |
| 查看品牌 / 产品详情 | 是 | 是 | 受限读取 | 否 | 否 |

---

## 9. 接口动作映射

| 页面动作 | 接口 / 命令语义 | 请求关键字段 | 成功反馈 | 失败反馈 |
|----------|-----------------|--------------|----------|----------|
| 新建品牌 | CreateBrand | brand_name | 返回 brand_id | 表单错误 / 权限不足 |
| 更新品牌 | UpdateBrandProfile | brand_id | 保存成功 | 字段校验失败 |
| 新建产品 | CreateProduct | brand_id, product_name, sku_code | 返回 product_id | 字段错误 |
| 更新产品 | UpdateProductProfile | product_id | 保存成功 | 权限或字段错误 |

---

## 10. 审计与追踪要求

关键写操作需记录品牌 ID、产品 ID、操作者角色、请求 ID、修改前后字段摘要。

---

## 11. 异常流

### 异常流 1

- 触发条件：品牌名称重复或违反唯一约束
- 页面表现：表单错误提示并阻止提交
- 用户下一步：修改品牌名称后重试
- 是否需要人工兜底：否

### 异常流 2

- 触发条件：Brand 试图编辑不属于本租户的数据
- 页面表现：提示无权限
- 用户下一步：联系平台处理租户边界问题
- 是否需要人工兜底：是

### 异常流 3

- 触发条件：产品字段不完整，无法进入试点投产
- 页面表现：详情页展示缺失项提示
- 用户下一步：补齐字段
- 是否需要人工兜底：否

---

## 12. 不做什么

- 不做产品发布审批流引擎
- 不做复杂多语言内容管理
- 不做营销素材中心

---

## 13. 验收标准（DoD）

### 13.1 产品验收

- Brand 能独立创建品牌与产品
- Platform 能查看并检索品牌 / 产品信息
- Factory 能读取必要产品摘要

### 13.2 联调验收

- 品牌 / 产品记录可被后续批次与激活流程正确引用
- 所有创建与更新写操作有审计记录

### 13.3 演示验收

- 可在试点演示中完成从品牌录入到产品建立的完整起点动作

---
