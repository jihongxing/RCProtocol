# 商业场景与买家

> 文档类型：Product  
> 状态：Active  
> 权威级别：Execution Baseline

---

## 1. 一句话商业价值

把高价值实物品牌当前分散、低效、不可追踪的“验真 + 激活 + 售后确权 + 二手流转治理”流程，收敛成一套可交付、可收费、可复用的标准化试点包，帮助品牌先解决真假争议、售后失控与二手渠道失控问题。

---

## 2. 第一目标行业

### 2.1 当前优先行业结论

第一阶段不做泛行业扩张，优先聚焦：

**具备现象级爆款、二手流转活跃、假货与渠道争议明显、品牌方有强控制诉求的高价值消费品品牌。**

优先顺序建议：

1. 现象级潮玩 / 新潮牌 IP 爆款
2. 高端酒类 / 限量酒款
3. 奢侈皮具 / 高端配饰

### 2.2 为什么不做泛行业

原因：

- 行业差异会导致交付成本上升
- 首单阶段最重要的是验证成交模型，不是扩场景
- 一人公司不能同时服务多个复杂行业

### 2.3 第一目标行业筛选标准

目标行业必须同时满足：

- 单件价值高
- 假货 / 调包 / 渠道争议真实存在
- 二级流转真实活跃，而不是只有展示没有交易
- 品牌方对售后关系有强诉求
- 标签增加的成本可被接受
- 品牌愿意推动试点流程变更

### 2.4 首个可成交切口

当前最适合切入的首个可成交场景，不是传统奢侈品总部系统改造，而是：

**拉布布类现象级潮玩 / 同类新潮牌爆款的验真、确权与二手流转试点。**

原因：

- 爆款单件价格已被二级市场拉高，真假争议与线下假货问题真实存在
- 用户已经形成“抢购、展示、转卖、交换”的资产化行为，只是品牌方尚未拿回入口
- 新潮牌团队对新机制、新 App、新玩法的接受度通常高于传统奢侈品牌
- 单件绝对价值虽低于奢侈皮具，但交易频次更高，更容易先验证流转模型
- 先打穿潮玩 / 新潮牌，再向奢侈皮具推进，组织阻力与教育成本更低

## 3. 第一付费买家是谁

### 3.1 付费买家定义

第一付费买家不是终端消费者，而是：

**品牌总部中能够同时对预算、品牌风控和售后治理负责的人。**

优先买家角色：

1. 品牌负责人 / 创始人
2. 品牌数字化负责人
3. 品牌渠道与售后负责人

### 3.2 为什么不是消费者

原因：

- 消费者虽能获得价值，但不是当前预算拥有者
- 当前阶段不能依赖 C 端规模化付费成立商业模型
- 应先证明 B 端 ROI，再讨论 C 端增值收入

### 3.3 为什么不是 IT 负责人

原因：

- IT 往往不是痛点最强的人
- 这个项目先卖的是“品牌控制力和售后治理结果”，不是“技术架构先进”
- 如果没有业务 owner 站台，项目会变成漫长 PoC

---

## 4. 买家核心痛点

第一买家不是因为“技术很酷”而购买，而是因为当前已经有持续损失。

### 4.1 当前必须解决的痛点

| 痛点 | 当前损失 | 为什么愿意付费 |
|---|---|---|
| 假货 / 调包争议 | 品牌声誉受损、售后争议成本高 | 需要一个可解释的验证机制 |
| 渠道窜货 / 来源不清 | 无法判断货从哪里流出 | 需要更强的单件追踪能力 |
| 售出后关系断裂 | 卖完后拿不到用户与资产后续信息 | 需要建立售后数字关系 |
| 门店 / 客服验证成本高 | 真假争议处理全靠人工经验 | 需要标准化验证路径 |
| 高价值商品售后难公证 | 用户和品牌各执一词 | 需要形成可审计事实链 |

### 4.2 不是当前核心痛点的内容

以下内容可以保留为远期价值，但不是第一单成交核心：

- C 端社区与增长玩法
- 二级流转生态平台
- 泛金融化故事
- 大规模会员化运营

---

## 5. 为什么客户现在就该买

### 5.1 当前购买理由

客户现在购买，不是为了“未来想象力”，而是为了立刻解决以下问题：

1. 当前真假争议已经影响品牌信任
2. 高价值商品售后处理成本过高
3. 品牌缺乏售后与持有关系的数据入口
4. 试点成本相对可控，能先在单 SKU 验证

### 5.2 立即购买的触发条件

以下任一条件成立，就具备强成交可能：

- 最近出现假货或掉包争议
- 品牌准备推高单价限量款
- 品牌正在升级数字化或售后体系
- 创始人 / 品牌负责人希望掌握售后关系而不是完全交给渠道

### 5.3 不应该卖给谁

以下对象不应作为第一批客户：

- 超低客单价快消品牌
- 没有品牌总部决策能力的代运营商
- 只想听概念、不愿意改任何流程的客户
- 只想做营销玩法、不在乎验真和售后治理的客户

## 6. 首个试点包卖什么

### 6.1 试点包定义

第一阶段不卖“完整平台”，只卖：

**单品牌、单 SKU、小批量标签试点包。**

### 6.2 推荐试点包内容

| 项目 | 试点范围 |
|---|---|
| 品牌数 | 1 |
| SKU 数 | 1 |
| 标签量 | 500 ~ 2000 |
| 后台 | 品牌 / 产品管理、盲扫、激活、审计 |
| C 端 | 验真页、基础资产详情、基础 Vault |
| 服务 | 配置、培训、试点陪跑、问题处理 |

### 6.3 试点包不包含的内容

| 不包含项 | 原因 |
|---|---|
| 完整二级流转市场 | 不是首单成交必要条件 |
| 高级委托和复杂审批 | 会显著拉高交付复杂度 |
| 深度 BI 报表 | 试点期可人工导出 |
| 全量多组织治理 | 首个品牌不需要 |

### 6.4 首个试点一页纸

建议把首个试点直接收敛成下面这类方案：

| 项目 | 建议 |
|---|---|
| 目标客户 | 拉布布类现象级潮玩或同类新潮牌爆款品牌 |
| 目标 SKU | 1 个最易出现假货与溢价流转的热门款 |
| 首批标签量 | 1000 ~ 3000 |
| 首批周期 | 2 ~ 4 周 |
| 首要问题 | 真假争议、线下假货、品牌无法掌握二手流转入口 |
| 交付能力 | 盲扫、激活、合法售出、持有入馆、转移登记、验真查询 |
| 品牌直接收益 | 官方掌握首批资产 ID、售后关系、二手流转记录与品牌触达入口 |
| 用户直接收益 | 获得官方可验证身份、持有展示、后续转移记录与更强掌控感 |
| 平台直接收益 | 接入费、年费、激活收入，以及后续流转确权费分成的验证基础 |

这个试点的销售话术应收敛为：

1. 先帮品牌拿回爆款二手流转入口，而不是先做完整平台改造。
2. 先把“真假争议 + 售后确权 + 二手登记”跑通，而不是先做完整交易市场。
3. 先在 1 个热门款上验证复购和转卖链路，再决定是否扩 SKU、扩渠道、扩品牌。

## 7. 如何定价与成交

### 7.1 第一阶段定价方式

第一阶段推荐只保留两段式收费：

1. **品牌试点接入费**
2. **按激活件数或批次收费**

不建议第一阶段同时引入复杂的：

- 流转分成
- 价值比例抽成
- 高级增值包

### 7.2 为什么用两段式收费

原因：

- 简单，容易理解
- 容易签单
- 容易和交付范围绑定
- 容易做试点结算

### 7.3 首单成交话术核心

首单销售不讲“大叙事”，只讲三件事：

1. 你可以先拿一个 SKU 做试点
2. 我们帮你把防伪、激活、验真和售后关系跑通
3. 你只需付试点接入费和激活执行费，不必先上完整平台

## 8. 客户 ROI 怎么讲

### 8.1 ROI 讲法原则

禁止只讲“品牌升级”“数字化转型”这类空话，必须讲：

- 节省了什么成本
- 降低了什么风险
- 增加了什么控制力
- 为后续复购 / 售后留下了什么入口

### 8.2 第一阶段 ROI 叙事

| ROI 维度 | 客户能获得什么 |
|---|---|
| 风险控制 | 减少真假争议与售后扯皮 |
| 品牌控制 | 建立品牌可控的激活和售出关系 |
| 运营效率 | 降低客服 / 门店人工判断成本 |
| 数据入口 | 拿到真实验真与售后触达入口 |
| 试点可控 | 先从一个 SKU 小规模验证，不必一次性大投入 |

### 8.3 不应承诺的 ROI

以下内容当前不应作为首单承诺：

- 直接提升 GMV
- 一定降低多少假货比例
- 一定带来多少复购率
- 立刻形成二级交易市场

这些结论需要试点数据，而不是销售时过度承诺。

## 9. 客户为什么会拒绝

### 9.1 主要拒绝原因

| 拒绝理由 | 本质问题 |
|---|---|
| 太复杂，流程改不动 | 交付阻力过大 |
| 我们先看看市场再说 | 痛点不够刚性或决策人不够强 |
| 预算有限 | 没有把试点包压到可接受范围 |
| 我们只想做营销，不想改售后流程 | 不是当前目标客户 |
| 需要多个部门一起决定 | 决策链太长，不适合首批成交 |

### 9.2 应对原则

- 不要为了成交去扩大范围
- 不要为了成交把 MVP 变成定制项目
- 如果客户必须要求多个非 MVP 功能，优先放弃，而不是接成一个高复杂度项目

---

## 10. 成交条件与准入标准

### 10.1 可以推进试点的客户

必须满足至少 4 项：

- 有明确高价值 SKU
- 能接受标签接入
- 有品牌总部 owner
- 有试点预算
- 能安排工厂或仓储流程配合
- 愿意给出 2 ~ 4 周试点窗口

### 10.2 不应推进的客户

以下客户应谨慎甚至直接放弃：

- 想一次性要完整平台
- 没有明确 owner
- 预算不清晰
- 无法协调工厂 / 仓储配合
- 把项目当纯 PPT 概念验证

---

## 11. 销售与交付的最小闭环

首个品牌试点必须收敛成以下闭环：

1. 签下 1 个品牌
2. 上线 1 个 SKU
3. 导入 500 ~ 2000 枚标签
4. 跑通盲扫 -> 激活 -> 售出 -> 验真 -> 入馆
5. 输出试点结果报告
6. 以结果推动续费或扩大 SKU

如果做不到这 6 步，就说明方案还不够锋利，或者客户筛选错了。

---

## 12. 当前结论

RCProtocol 第一阶段不应该卖成“宏大的资产主权平台”，而应该卖成：

**一个帮助高价值品牌快速验证“可控激活 + 官方确权 + 售后可追踪 + 二手流转可治理”的低风险试点包。**

这才是当前最容易成交、最容易交付、也最容易验证 ROI 的商业路径。
